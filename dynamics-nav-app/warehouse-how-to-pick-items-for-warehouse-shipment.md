# <a name="how-to-pick-items-for-warehouse-shipment"></a><span data-ttu-id="892b5-103">Практическое руководство: подбор товаров для складской отгрузки</span><span class="sxs-lookup"><span data-stu-id="892b5-103">How to: Pick Items for Warehouse Shipment</span></span>
<span data-ttu-id="892b5-104">Если склад настроен так, что требуется обработка как складского подбора, так и складской отгрузки, то для создания и обработки информации о подборе до учета складской отгрузки используются документы складского подбора.</span><span class="sxs-lookup"><span data-stu-id="892b5-104">When the location is set up to require warehouse pick processing as well as warehouse shipment processing, you use the warehouse pick documents to create and process pick information prior to posting the warehouse shipment.</span></span>  

<span data-ttu-id="892b5-105">Нельзя создать документ складского подбора с нуля, поскольку операция подбора всегда является частью рабочего процесса как в сценарии извлечения, так и в сценарии распределения.</span><span class="sxs-lookup"><span data-stu-id="892b5-105">You cannot create a warehouse pick document from scratch because a pick activity is always part of a workflow, either in a pull or a push scenario.</span></span>  

<span data-ttu-id="892b5-106">Можно создать документы складского подбора типа push, открыв пустой документ складской отгрузки, обнаружив документы-источники, выпущенные для отгрузки, а затем создав строки складского подбора для этих отгрузок.</span><span class="sxs-lookup"><span data-stu-id="892b5-106">You can create warehouse pick documents in a pull fashion by opening an empty warehouse shipment document, detect source documents that are released to shipment, and then create warehouse pick lines for those shipments.</span></span> <span data-ttu-id="892b5-107">Можно использовать функцию **Получить исходные документы** или **Использовать фильтр для получения исходных документов** для выявления документов-источников, которые готовы к отгрузке.</span><span class="sxs-lookup"><span data-stu-id="892b5-107">You can use the **Get Source Documents** or **Use Filter to Get Source Documents** functions to detect source documents that are ready for shipment.</span></span>

<span data-ttu-id="892b5-108">Кроме того, можно использовать окно **Журнал подбора** для извлечения и создания строк подбора в пакетном режиме.</span><span class="sxs-lookup"><span data-stu-id="892b5-108">Alternatively, you can use the **Pick Worksheet** window to pull and create pick lines in batch mode.</span></span> <span data-ttu-id="892b5-109">Дополнительные сведения см. в разделе [Практическое руководство. Планирование подборов в журналах](warehouse-how-to-plan-picks-in-worksheets.md).</span><span class="sxs-lookup"><span data-stu-id="892b5-109">For more information, see [How to: Plan Picks in Worksheets](warehouse-how-to-plan-picks-in-worksheets.md).</span></span>  

<span data-ttu-id="892b5-110">Можно также создать документы складского типа push в окне **Складская отгрузка** нажатием **Создать подбор**.</span><span class="sxs-lookup"><span data-stu-id="892b5-110">You can also create warehouse pick documents in a push fashion from the **Warehouse Shipment** window by selecting **Create Pick**.</span></span>  

> [!NOTE]  
>  <span data-ttu-id="892b5-111">Подбор для складской отгрузки товаров, которые собираются для заказа на продажу, выполняется так же, как для обычной отгрузки для складских подборов, как описано в этом разделе.</span><span class="sxs-lookup"><span data-stu-id="892b5-111">Picking for warehouse shipment of items that are assembled to the sales order being shipped follows the same steps as for regular warehouse picks for shipment, as described in this topic.</span></span> <span data-ttu-id="892b5-112">Количество строк подбора на количество для отгрузки может равняться "один ко многим", так как вы выбираете компоненты, а не сборочный элемент.</span><span class="sxs-lookup"><span data-stu-id="892b5-112">However, the number of pick lines per quantity to ship may be many to one because you pick the components, not the assembly item.</span></span>  
>   
>  <span data-ttu-id="892b5-113">Строки складского подбора создаются для значения в поле **Остаток** в строках сборочного заказа, связанного со строкой заказа на продажу, который отгружается.</span><span class="sxs-lookup"><span data-stu-id="892b5-113">The warehouse pick lines are created for the value in the **Remaining Quantity** field on the lines of the assembly order that is linked to the sales order line that is being shipped.</span></span> <span data-ttu-id="892b5-114">Это гарантирует, что все компоненты подбираются одним действием.</span><span class="sxs-lookup"><span data-stu-id="892b5-114">This ensures that all components are picked in one action.</span></span>  
>   
>  <span data-ttu-id="892b5-115">Для получения дополнительных сведений см. раздел "Обработка сборок на заказ в складских отгрузках".</span><span class="sxs-lookup"><span data-stu-id="892b5-115">For more information, see the “Handling Assemble-to-Order Items in Warehouse Shipments” section.</span></span>  
>   
>  <span data-ttu-id="892b5-116">Для получения информации о подборе компонентов для сборочных заказов, в том числе о ситуациях когда сборочный элемент не должен быть в расходной накладной, см. раздел [Практическое руководство. Подбор для производства или сборки](warehouse-how-to-pick-for-production.md).</span><span class="sxs-lookup"><span data-stu-id="892b5-116">For information about picking components for assembly orders generally, including situations where the assembly item is not due on a sales shipment, see [How to: Pick for Production or Assembly](warehouse-how-to-pick-for-production.md).</span></span>  

## <a name="to-pick-items-for-warehouse-shipment"></a><span data-ttu-id="892b5-117">Складской подбор товаров для отгрузки</span><span class="sxs-lookup"><span data-stu-id="892b5-117">To pick items for warehouse shipment</span></span>  
1.  <span data-ttu-id="892b5-118">Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Складской подбор**, затем выберите связанную ссылку.</span><span class="sxs-lookup"><span data-stu-id="892b5-118">Choose the ![Search for Page or Report](media/ui-search/search_small.png "Search for Page or Report icon") icon, enter **Picks**, and then choose the related link.</span></span>  

    <span data-ttu-id="892b5-119">Если необходимо работать с определенным подбором, выберите этот подбор в списке или отфильтруйте список, чтобы найти подборы, назначенные для вас.</span><span class="sxs-lookup"><span data-stu-id="892b5-119">If you need to work on a particular pick, select the pick from the list or filter the list to find the picks that have been assigned to you specifically.</span></span> <span data-ttu-id="892b5-120">Откройте карточку подбора.</span><span class="sxs-lookup"><span data-stu-id="892b5-120">Open the pick card.</span></span>  
2.  <span data-ttu-id="892b5-121">Если поле **Код исполнителя** пустое, при необходимости введите для идентификации свой код.</span><span class="sxs-lookup"><span data-stu-id="892b5-121">If the **Assigned User ID** field is empty, enter your ID to identify yourself if necessary.</span></span>  
3.  <span data-ttu-id="892b5-122">Выполнить фактический подбор товаров.</span><span class="sxs-lookup"><span data-stu-id="892b5-122">Perform the actual picking of items.</span></span>  

    <span data-ttu-id="892b5-123">Если склад настроен на использование ячеек, то ячейки товаров по умолчанию используются для того, чтобы предложить, откуда взять товары.</span><span class="sxs-lookup"><span data-stu-id="892b5-123">If the warehouse is set up to use bins, then the items’ default bins are used to suggest where to take the items from.</span></span> <span data-ttu-id="892b5-124">Инструкции отображаются в виде двух раздельных строк, не менее одной для каждого типа действий "Взять" и "Поместить".</span><span class="sxs-lookup"><span data-stu-id="892b5-124">The instructions appear as two separate lines, at least one for each action type, Take and Place.</span></span>  

    <span data-ttu-id="892b5-125">Если склад настроен на использование расширенного подбора и размещения, для вычисления наиболее подходящих для подбора ячеек используется рейтинг ячеек; затем такие ячейки предлагаются в строках подбора.</span><span class="sxs-lookup"><span data-stu-id="892b5-125">If the warehouse is set up to use directed put-away and pick, then the bin ranking is used to calculate the best bins to pick from, and those bins are then suggested on the pick lines.</span></span> <span data-ttu-id="892b5-126">Инструкции отображаются в виде двух раздельных строк, не менее одной для каждого типа действий "Взять" и "Поместить".</span><span class="sxs-lookup"><span data-stu-id="892b5-126">The instructions appear as two separate lines, at least one for each action type, Take and Place.</span></span>  

4.  <span data-ttu-id="892b5-127">После выполнения подбора и помещения товаров в зону отгрузки или ячейку отгрузки выберите действие **Зарегистрировать подбор**.</span><span class="sxs-lookup"><span data-stu-id="892b5-127">When you have performed the pick and placed the items in the shipping area or shipping bin, choose the **Register Pick** action.</span></span>  

<span data-ttu-id="892b5-128">Лицо, ответственное за отгрузку, теперь может перенести товары в отгрузочный отсек и учесть отгрузку, включая соответствующий документ-источник, в окне **Складская отгрузка**.</span><span class="sxs-lookup"><span data-stu-id="892b5-128">The person responsible for shipment can now bring the items to the shipment dock and post the shipment, including the related source document, in the **Warehouse Shipment** window.</span></span> <span data-ttu-id="892b5-129">Дополнительные сведения см. в разделе [Практическое руководство. Отгрузка товаров](warehouse-how-ship-items.md).</span><span class="sxs-lookup"><span data-stu-id="892b5-129">For more information, see [How to: Ship Items](warehouse-how-ship-items.md).</span></span>   

<span data-ttu-id="892b5-130">В дополнение к подбору для документов-источников, как описано в этом разделе, можно взять и поместить товары между ячейками без обращения к документам-источникам.</span><span class="sxs-lookup"><span data-stu-id="892b5-130">In addition to picking for source documents, as described in this topic, you can take and place items between bins without referring to source documents.</span></span> <span data-ttu-id="892b5-131">Дополнительные сведения см. в разделе [Практическое руководство. Подбор и размещение без документа-источника](warehouse-how-to-create-put-aways-from-internal-put-aways.md).</span><span class="sxs-lookup"><span data-stu-id="892b5-131">For more information, see [How to: Pick and Put Away Without a Source Document](warehouse-how-to-create-put-aways-from-internal-put-aways.md).</span></span>  

## <a name="handling-assemble-to-order-items-in-warehouse-shipments"></a><span data-ttu-id="892b5-132">Обработка товаров сборки на заказ складских товаров в складских отгрузках</span><span class="sxs-lookup"><span data-stu-id="892b5-132">Handling Assemble-to-Order Items in Warehouse Shipments</span></span>
<span data-ttu-id="892b5-133">В сценариях сборки на заказ, поле **Кол-во для отгрузки** в строках складской отгрузки используется для записи количества собранных единиц.</span><span class="sxs-lookup"><span data-stu-id="892b5-133">In assemble-to-order scenarios, the **Qty. to Ship** field on warehouse shipment lines is used to record how many units are assembled.</span></span> <span data-ttu-id="892b5-134">Затем указанное количество учитывается как производительность сборки при учете расходной накладной.</span><span class="sxs-lookup"><span data-stu-id="892b5-134">The specified quantity is then posted as assembly output when the warehouse shipment is posted.</span></span>

<span data-ttu-id="892b5-135">Для других строк учтенной складской отгрузки значение в поле **Кол-во для отгрузки** является нулевым с самого начала.</span><span class="sxs-lookup"><span data-stu-id="892b5-135">For other warehouse shipment lines, the value in the **Qty. to Ship** field is zero from the start.</span></span>

<span data-ttu-id="892b5-136">Когда работники, ответственные за сборку, заканчивают собирать часть или все элементы сборки на заказ, они регистрируют ее в поле **Кол-во для отгрузки** в строке складской отгрузки и выбирают действие **Учет отгрузки**.</span><span class="sxs-lookup"><span data-stu-id="892b5-136">When workers in charge of assembly finish assembling parts or all of the assemble-to-order quantity, they record it in the **Qty. to Ship** field on the warehouse shipment line and then choose the **Post Shipment** action.</span></span> <span data-ttu-id="892b5-137">Результат заключается в том, что учитывается соответствующая производительность сборки, включая потребление компонентов.</span><span class="sxs-lookup"><span data-stu-id="892b5-137">The result is that the corresponding assembly output is posted, including the component consumption.</span></span> <span data-ttu-id="892b5-138">Расходная накладная для количества учитывается для заказа на продажу.</span><span class="sxs-lookup"><span data-stu-id="892b5-138">A sales shipment for the quantity is posted for the sales order.</span></span>

<span data-ttu-id="892b5-139">Из сборочного заказа можно выбрать **Строка Отгрузка со склада сборки на заказ** для доступа к строкам складской отгрузки.</span><span class="sxs-lookup"><span data-stu-id="892b5-139">From the assembly order, you can choose **Asm.-to-Order Whse. Line** to access the warehouse shipment line.</span></span> <span data-ttu-id="892b5-140">Это удобно для работников, которые обычно не используют окно **Складская отгрузка**.</span><span class="sxs-lookup"><span data-stu-id="892b5-140">This is convenient for workers who do not typically use the **Warehouse Shipment** window.</span></span>

<span data-ttu-id="892b5-141">После учета расходной накладной различные поля в строке заказа на продажу обновляются, отображая продвижение работы по складу.</span><span class="sxs-lookup"><span data-stu-id="892b5-141">After the warehouse shipment is posted, various fields on the sales order line are updated to show progress in the warehouse.</span></span> <span data-ttu-id="892b5-142">Следующие поля также обновляются, чтобы показать объем сборки на заказ, оставшийся для сборки и отгрузки.</span><span class="sxs-lookup"><span data-stu-id="892b5-142">The following fields are also updated to show how many assemble-to-order quantities remain to be assembled and shipped:</span></span>

- <span data-ttu-id="892b5-144">**Недопоставленное количество по сборке на заказ на складе (базовое)**</span><span class="sxs-lookup"><span data-stu-id="892b5-144">**ATO Whse. Outstd. Qty. (Base)**</span></span>

> [!NOTE]
> <span data-ttu-id="892b5-145">В составных сценариях, согласно которым часть количества нужно сначала собрать, а другую отгрузить из запасов, создаются две строки складской отгрузки.</span><span class="sxs-lookup"><span data-stu-id="892b5-145">In combination scenarios, in which a part of the quantity must first be assembled and another must be shipped from inventory, two warehouse shipment lines are created.</span></span> <span data-ttu-id="892b5-146">Одно значение для количества для сборки на заказ, а другое для количества запасов.</span><span class="sxs-lookup"><span data-stu-id="892b5-146">One is for the assemble-to-order quantity, and one is for the inventory quantity.</span></span>
