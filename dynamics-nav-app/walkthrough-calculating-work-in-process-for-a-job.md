# <a name="walkthrough-calculating-work-in-process-for-a-job"></a>Пошаговое руководство. Вычисление незавершенного производства для задания
С помощью работ можно планировать использование ресурсов организации и отслеживать различные затраты, связанные с использованием ресурсов в конкретном проекте. В приложении "Работы" учитывается использование человеко-часов, станко-часов, товаров в запасах и других типов потребления, которые необходимо отслеживать по мере выполнения работы. Если работа выполняется в течение длительного периода, возможно, эти затраты потребуется переносить на счет незавершенного производства в балансе по мере выполнения работы. Затем при необходимости можно признать затраты и продажи в счетах отчета о прибыли и убытках.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве  
 В этом пошаговом руководстве рассматриваются следующие задачи:  

-   вычисление незавершенного производства;  
-   выбор метода вычисления незавершенного производства;  
-   Исключение части работы из незавершенного производства.  
-   Учет НЗП в Главной книге.  
-   сторнирование учета незавершенного производства.  

 На каждом этапе процесса вычисляется значение, а транзакции работы перемещаются в главную книгу. Этап вычисления и учета выполняются отдельно друг от друга, что позволяет просматривать данные и вносить изменения перед учетом в главной книге. Поэтому после выполнения пакетных заданий по вычислению перед запуском пакетных заданий по учету следует убедиться в правильности всей информации.  

## <a name="roles"></a>Роли  
 В этом пошаговом руководстве в роли действующего лица выступает член проектной группы (Татьяна).  

## <a name="prerequisites"></a>Предварительные требования  
 Перед выполнением заданий этого пошагового руководства необходимо установить на компьютере [!INCLUDE[d365fin](includes/d365fin_md.md)].  

## <a name="story"></a>Сюжет  
 Действия этого пошагового руководства разворачиваются вокруг организации «CRONUS International Ltd.», консультационно-конструкторской фирмы, занимающейся проектировкой и монтажом новых инфраструктур, например конференц-залов и офисов с мебелью, вспомогательным оснащением и камерой хранения. Большинство работ в CRONUS ориентированы на проекты, и Татьяна, член проектной группы, использует задания, чтобы получить представление о каждой выполняемой работе, начатой CRONUS, а также о завершенных работах. Некоторые из заданий могут быть очень продолжительными и выполняться месяцами. Татьяна может использовать счет НЗП для записи НЗП и отслеживать себестоимости в процессе работы.  

## <a name="calculating-wip"></a>Вычисление незавершенного производства  
 Организация CRONUS взялась за длительный проект, который растянется на несколько отчетных периодов. Член проектной группы Татьяна рассчитывает объем незавершенного производства (НЗП), чтобы убедиться в точности финансового отчета организации.  

 В ходе этой процедуры Татьяна выберет определенную группу задач, для которых будет вычислено незавершенное производство. В окне **Строки рабочего задания** она может указать соответствующие строки в столбце **Итоги по НЗП**.  

 В следующей таблице описаны три параметра.  

|Поле|Описанием|  
|-------------------------------------|---------------------------------------|  
|**<blank>**|Оставьте пустым, если рабочее задание входит в группу задач.|  
|**Итого**|Определяет диапазон или группу задач, которые будут включены в расчет НЗП и признания. В любой группе рабочее задание, у которого для параметра **Тип рабочего задания** задано значение **Учет**, будет включено в поле "Итоги по НЗП", если только значение **Итоги по НЗП** не равно **Исключено**.|  
|**Исключено**|Применяется только к задаче, у которой **Тип рабочего задания** — **Учет**. Эта задача не включается при расчете НЗП и признания.|  

 В следующем пошаговом руководстве Татьяна применяет для метод себестоимости, стандарт ее компании, чтобы рассчитать НЗП. Она указывает, какая часть работы будет добавлена при вычислении незавершенных производственных работ, задавая значения НЗП для различных строк рабочего задания.  

### <a name="to-calculate-wip"></a>Расчет НЗП  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Работы**, а затем выберите связанную ссылку.  
2.  В списке **Работы** выберите работу **Графика**, затем выберите действие **Изменить**. Это приведет к открытию карточки работы в режиме правки.  

     Незавершенное производство можно вычислить на основе себестоимости, суммы продаж, себестоимости проданных товаров, процента выполнения или выполненного контракта. В этом примере CRONUS использует метод себестоимости.  

3.  На экспресс-вкладке **Учет** щелкните в поле **Метод НЗП** и выберите **Себестоимость**.  
4.  Выберите действие **Строки рабочих заданий** и установите следующие значения в поле **Итоги по НЗП**.  

     В следующей таблице приводится описание значений.  

    |Номер рабочего задания|Общее поле НЗП|  
    |------------------|----------------------|  
    |1130|Исключено|  
    |1190|Итого|  
    |1210|Исключено|  
    |1310|Исключено|  

5.  Выберите действие **НЗП**, затем выберите действие **Рассчитать НЗП**.  
6.  В окне **Работа по расчету НЗП** можно выбрать работу, для которой необходимо вычислить незавершенное производство. На экспресс\-вкладке **Работа** в поле **Номер** выберите значение **Графика**. .  
7.  В поле **Дата учета** введите дату позже рабочей даты.
8.  В поле **Номер документа** введите **1**. Это создает документ, на который можно в дальнейшем ссылаться для обеспечения возможности трассировки.  
9. Нажмите кнопку **OK** для запуска пакетного задания. Отобразится сообщение. Нажмите кнопку **OK**, чтобы продолжить. Закройте окно **Рабочее задание - строки**.  

    > [!NOTE]  
    >  Сообщение указывает, что имеются предупреждения, связанные с расчетом НЗП. В следующей процедуре вы просмотрите предупреждения.  

10. На карточке **Работа** разверните экспресс-вкладку **НЗП и призн. доходов и затрат**, чтобы просмотреть вычисленные значения. Можно также просмотреть **Дата учета НЗП** и значения, которые были учтены в главной книге, если таковые были.  

 Обратите внимание, что для поля **Призн. сумма затрат** в столбце **Учесть** выбрано значение 215,60. Оно отражает общую себестоимость двух товаров в группе рабочих заданий с 1110 по 1130. Для третьего товара выбрано значение **Исключено**, поэтому он не включен в расчет НЗП.  

### <a name="to-review-wip-warnings"></a>Обзор по предупреждениям НЗП  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Панель задач НЗП**, затем выберите связанную ссылку.  
2.  Выберите работу **Графика**, затем выберите действие **Показать предупреждения**.  
3.  В окне **Предупреждения о НЗП работ** просмотрите предупреждение, связанное с заданием.  

 После окончания учетного периода Татьяна должна рассчитать повторно незавершенное производство, чтобы включить в расчет завершенную на данный момент работу.  

### <a name="to-recalculate-wip"></a>Пересчет НЗП  

1.  На карточке **Работа** выберите действие **Операции НЗП** для просмотра расчета НЗП.  

     В окне **Работа - операции НЗП** отображаются последние вычисленные для работы операции незавершенного производства, даже если незавершенное производство еще не учтено в главной книге.  

2.  Выполните последовательность действий, объясняющих, как подсчитывать НЗП и пересчитывать НЗП. При каждом вычислении незавершенного производства в окне **Работа - операции НЗП** создается операция.  
3.  Закройте окно.  

> [!NOTE]  
>  Функция определения незавершенного производства и признания доходов и затрат предполагает только вычисление. Операция не учитывается в Главной книге. Для учета следует выполнить пакетное задание **Учет НЗП в ГК** после вычисления значения "НЗП и призн. доходов и затрат".

## <a name="posting-wip-to-general-ledger"></a>Учет незавершенного производства в главной книге  
 Теперь, после вычисления незавершенного производства для данного задания, Татьяна может учесть его в Главной книге.  

### <a name="to-post-wip-to-general-ledger"></a>Учет незавершенного производства в главной книге  

1.  В списке **Работы** выберите работу **Графика**.  
2.  Выберите действие **НЗП**, затем выберите действие **Учет НЗП в ГК**.  
3.  В окне **Работа - учет НЗП в ГК** на экспресс-вкладке **Работа** выберите в поле **Номер** значение **Графика**. .  
4.  На экспресс-вкладке **Параметры** в поле **Номер сторнированного документа** введите **1**.  
5.  Нажмите кнопку **OK** для учета НЗП в Главной книге.  
6.  Нажмите кнопку **OK**, чтобы закрыть окно подтверждения.  

     После завершения учета можно просмотреть учтенную информацию в окне **Операции НЗП в ГК**.  

7.  В списке **Работы** выберите работу **Графика**, затем выберите действие **Операции НЗП в ГК**.  

     В окне **Работа - операции ГК НЗП** убедитесь, что незавершенное производство учтено в Главной книге.  

8.  Закройте окно.  
9. Откройте карточку **Работа** для работы **Графика**.  
10. На экспресс-вкладке **НЗП и признание доходов и затрат** обратите внимание, что в столбце **Учтено** поле **Сумма признанных затрат ГК** теперь заполнена, следовательно НЗП учтено в Главной книге.  
11. Нажмите кнопку **OK**, чтобы закрыть карточку.  

## <a name="reversing-a-wip-posting"></a>Сторнирование учета незавершенного производства  
 Татьяна выясняет, что рабочие задания, исключенные из расчета суммы незавершенного производства, должны вычисляться в НЗП. Она может сторнировать неправильный учет, не учитывая новые данные по незавершенному производству.  

### <a name="to-reverse-a-wip-posting"></a>Процедура сторнирования учета незавершенного производства  

1.  В списке **Работы** выберите работу **Графика**.  
2.  Выберите действие **НЗП**, затем выберите действие **Учет НЗП в ГК**.  
3.  В окне **Работа - учет НЗП в ГК** на экспресс-вкладке **Работа** выберите в поле **Номер** значение **Графика**. .  
4.  На экспресс-вкладке **Параметры** в поле **Номер сторнированного документа** введите **1**.  
5.  В поле **Дата учета сторнирования** введите исходную дату учета. Это должна быть та же дата, которая использована, чтобы вычислить НЗП в первый раз.  
6.  Установите флажок **Только сторнировать**. Это приведет к сторнированию учтенного ранее незавершенного производства без учета нового НЗП в главной книге.  
7.  Нажмите кнопку **OK**, чтобы запустить пакетное задание, а затем нажмите кнопку **OK**, чтобы закрыть окно подтверждения.  
8.  Откройте карточку **Работа** для компании **Графика**.  
9. На экспресс-вкладке **НЗП и признание доходов и затрат** убедитесь, что нет учтенных операций НЗП.  
10. Закройте это окно.  
11. В списке **Работы** выберите работу **Графика**, выберите действие **НЗП**, затем выберите действие **Операции НЗП в ГК**. Для операций НЗП флажок **Сторнировано** установлен.  
12. Закройте это окно.  
13. Откройте страницу **Строки рабочего задания** для данной работы, включите части работы, которые должны быть в расчете НЗП, а затем выполните пересчет и учет нового значения в Главной книге.  

    > [!NOTE]  
    >  Предположим, Татьяна вычислила и учла незавершенное производство для работы с неправильными датами. Придерживаясь описанного ранее метода, она может сторнировать неправильные записи учета, исправить даты и выполнить повторный учет в Главной книге.
