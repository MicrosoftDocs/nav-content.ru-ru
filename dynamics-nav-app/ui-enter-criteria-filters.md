# <a name="searching-filtering-and-sorting-data"></a>Поиск, фильтрация и сортировка данных
Есть несколько вещей, которые могут помочь искать, определять местоположение и сканировать записи в списке. Сюда входят сортировка, поиск и фильтрация.

Если необходимо выполнить поиск данных, таких как название клиента, адреса или товарные группы, вводятся критерии. В критериях поиска можно использовать все цифры и буквы, которые вводятся в соответствующие поля. Кроме того, для дополнительной фильтрации результатов можно использовать специальные символы. Существует два способа поиска: с помощью экспресс-фильтра или фильтров столбцов.

## <a name="sorting"></a>Сортировка
Сортировка позволяет быстро и легко получить общее представление о данных. При наличии большого числа клиентов, например, можно сортировать их по полю **Номер клиента**, **Учетная группа клиента**, **Код валюты**, **Код страны/региона** или **ИНН**, чтобы получить нужный обзор.

Для сортировки списка можно либо выбрать текст заголовка столбца для переключения между порядком по возрастания или по убыванию, либо выбрать небольшую стрелку вниз в заголовке столбца, затем выбрать **По возрастанию** или **По убыванию**.  

> [!NOTE]  
>   Не поддерживается сортировка изображений, полей BLOB, полей FlowFilter и полей, которые не принадлежат таблице.  

## <a name="searching-by-using-the-quick-filter"></a>Поиск с помощью экспресс-фильтра
Вы можете добавить фильтры на все страницы с помощью экспресс-фильтра. Экспресс-фильтр включается при выборе значка лупы в правом верхнем углу страницы. Данный тип фильтра используется для быстрого ввода критериев.

> [!IMPORTANT]  
>   Экспресс-фильтр обеспечивает удобный доступ к фильтруемым данным путем ввода простого текста, а также множество вариантов указания критериев поиска. В зависимости от того, введен ли обычный текст или текст с символами, поведение экспресс-фильтра отличается.  

* Если в критериях поиска ввести обычный текст, критерии поиска будут рассматриваться без учета регистра и будет выполняться поиск определенного текста.  
* Если в критериях поиска ввести тест с символами, критерии поиска будут рассматриваться с учетом регистра и будет выполняться поиск точного текста.

>   Нельзя использовать подстановочные символы при фильтрации в полях перечислений, например в поле **Статус** заказов на продажу. Например, в поле **Статус** в заказе на продажу, которое имеет значения **Открыть**, **Выпущено**, **Ожидает утверждения** и **Ожидает предоплаты**, используйте значения **0**, **1**, **2**, и **3** для фильтрации этих параметров. 

## <a name="searching-by-using-column-filters"></a>Поиск с помощью фильтров столбцов
Можно добавить фильтр на один или несколько столбцов в списке. Фильтрация по столбцах обеспечивает большую гибкость и возможности, чем экспресс-фильтр. 

### <a name="to-add-a-filter-on-a-column"></a>Добавление фильтра в столбец
1.  Прежде чем добавлять фильтр, выберите значок ![Показать как список](media/ui_show_as_list_icon.png "Стрелка влево, показать как список"), чтобы переключиться в представление списка.
2. Выберите стрелку вниз в заголовке столбца, затем выберите **Фильтр**.
3. Выполните одно из следующих действий: 
  -  Выберите *...* рядом с полем для выбора значения из списка.
  -  Введите критерии фильтра в поле. Подробнее см. в следующем разделе.
4. Нажмите кнопку **ОК**.

## <a name="filter-criteria-and-symbols"></a>Критерии фильтра и символы
В процессе установки критериев фильтра можно использовать все цифры и буквы, которые обычно вводятся в поле. Кроме того, для дополнительной фильтрации результатов можно использовать специальные символы. В следующих таблицах приведены символы, которые могут быть использованы в фильтрах.  
  
> [!IMPORTANT]  
>  Могут быть ситуации, когда значения поля содержит эти символы, и требуется фильтрация по ним. Чтобы это сделать, необходимо включить выражение фильтра, которое содержит данный символ в кавычках ("). Например, если необходимо фильтровать записи, которые начинаются с текста *S&R*, выражение фильтра имеет вид **'S&R*'**.  
  
### <a name="-interval"></a>(..) Диапазон  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|1 100..2 100|Номера от 1 100 до 2 100|  
|..2 500|До 2500, включительно|  
|..31.12.00|Даты по 31.12.00 включительно|  
|P8..|Информация для 8-го учетного периода и позже.|  
|..23|С даты начала по 23 число текущего месяца текущего года до 23:59:59|  
|23..|с 23 текущего месяца текущего года 00:00:00 до конца|  
|22..23|с 22 текущего месяца текущего года 0:00:00 по 23 текущего месяца текущего года 23:59:59|  
  
### <a name="124-eitheror"></a>(&#124;) Либо/или  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|1200&#124;1300|Номера со значением 1 200 или 1 300|  
  
### <a name="-not-equal-to"></a>(<>) Не равно  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|<>0|Все числа, кроме 0<br /><br /> Возможность использования Microsoft SQL Server позволяет комбинировать данный символ со знаками подстановки. Например, <>A* означает несоответствие любым текстам, начинающимся с А.|  
  
### <a name="-greater-than"></a>(>) Больше чем  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|>1 200|Числа больше, чем 1 200|  
  
### <a name="-greater-than-or-equal-to"></a>(>=) Больше чем или равно  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|>=1200|Номера больше или равны 1 200|  
  
### <a name="-less-than"></a>(<) Меньше чем  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|<1200|Номера меньше, чем 1 200|  
  
### <a name="-less-than-or-equal-to"></a>(<=) Меньше или равно  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|<=1 200|Номера меньше или равны 1 200|  
  
### <a name="-and"></a>(&) И  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|>200&<1200|Числа больше 200 и меньше 1200|  
  
### <a name="-an-exact-character-match"></a>('') Точное совпадение символа  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|'man'|Текст, точно соответствующий man с учетом регистра.|  
  
### <a name="-case-insensitive"></a>(@) Без учета регистра  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|@man*|Текст, начинающийся с man без учета регистра.|  
  
### <a name="-an-indefinite-number-of-unknown-characters"></a>(*) Неопределенное количество неизвестных символов  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|*Ко*|Текст, который содержит "Ко" с учетом регистра.|  
|*Ко|Текст, который заканчивается на "Ко" с учетом регистра.|  
|Ко*|Текст, который начинается с "Ко" с учетом регистра.|  
  
### <a name="-one-unknown-character"></a>(?) Один неизвестный символ  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|Ст?л|Такой текст, как «Стул» или «Стол»|  
  
### <a name="combined-format-expressions"></a>Объединенные выражения форматов  
  
|Образец выражения|Отображаемые записи|  
|-----------------------|-----------------------|  
|5999&#124;8100..8490|Включает все записи с номером 5999 или номерами от 8100 до 8490.|  
|..1299&#124;1400..|Все записи с номером меньше или равно 1299 или номером от 1400 или больше (все номера, кроме от 1300 до 1399).|  
|>50&<100|Все записи с номерами больше 50 и меньше 100 (номера от 51 до 99).|
