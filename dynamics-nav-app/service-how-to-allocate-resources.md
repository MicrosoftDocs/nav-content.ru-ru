# <a name="how-to-allocate-resources"></a>Практическое руководство. Распределение ресурсов
Ключевым элементом сервисного управления является персонал, предоставляющий услуги. Можно настроить [!INCLUDE[d365fin](includes/d365fin_md.md)] для назначения правильных сотрудников соответствующим работам. Назначения могут задаваться на основе сервисных зон, в которых находятся сотрудники, или мест, где производится сервисное обслуживание. Кроме этого, при реагировании на сервисные запросы можно группировать ресурсы. Дополнительные сведения см. в разделе [Практическое руководство. Настройка распределения ресурсов](service-how-setup-resource-allocation.md).

Может распределять ресурсы, например технических специалистов, с помощью страницы **Панель диспетчера** или из сервисного заказа. Можно использовать доступность ресурсов для распределения ресурсов для выполнения сервисных задач в заказах или предложениях.

Один и тот же ресурс, например специалиста, или одну группу ресурсов можно распределить для всех сервисных товаров сервисного заказа. Операции распределения создаются для других сервисных товаров в заказе с теми же номером ресурса и датой распределения, что и у распределенной строки. Распределённые часы – это частное от деления количества часов, которые были распределены, на число сервисных товаров в заказе. Поле **Статус** автоматически принимает значение **Активно** для всех операций, которые были созданы.

## <a name="to-see-an-overview-of-service-orders-and-service-quotes"></a>Просмотр обзора сервисных заказов и предложений  
Зачастую может возникать необходимость просмотреть список сервисных заказов или сервисных предложений, отвечающих определенным условиям, чтобы выполнить над ними определенные действия. Например, может возникнуть необходимость распределить ресурсы на сервисные заказы, принадлежащие определенному клиенту.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Панель диспетчера**, затем выберите связанную ссылку.  
2. В поле **Фильтр по документу** выберите тип для документов, которые хотите просмотреть.
3. Чтобы получить список документов с сервисными задачами, на которые выделен определенный ресурс или группа ресурсов, заполните поля **Фильтр по ресурсу** и **Фильтр по группе ресурсов**, затем нажмите клавишу ВВОД.  
4. Чтобы получить список документов, которым назначена определенная дата отклика, или даты отклика которых оказываются в пределах некоторого временного промежутка, заполните поле **Фильтр по дате отклика** и нажмите клавишу **ВВОД**.  
5. Чтобы получить список документов с определенным состоянием распределения или статусом документа, заполните поле **Фильтр распределения/фильтр состояния** и нажмите клавишу **ВВОД**.  
6. Чтобы получить список документов, относящихся к определенному контракту, клиенту или зоне, заполните поля **Фильтр контракта/фильтр клиента/фильтр зоны** и нажмите клавишу **ВВОД**.  
7. Выберите строку, которая соответствует сервисному заказу или сервисному предложению, затем выберите действие **Показать документ**.  

    Откроется страница **Сервисный заказ** или **Сервисное предложение**, и можно будет работать с документом. Чтобы вернуться на страницу **Панель диспетчера**, нажмите кнопку **OK**.

## <a name="to-allocate-a-resource-using-resource-or-resource-group-availability"></a>Распределение ресурса на основе доступности ресурса или группы ресурсов    
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Панель диспетчера**, и выберите связанную ссылку.  
2. Выберите сервисный заказ, затем выберите действие **Распределение ресурсов**.  
3. Выберите операцию, содержащую сервисную задачу, которой предполагается распределить данный ресурс.  
4. Выберите действие **Наличие ресурсов** или **Наличие группы ресурсов**.  
5. В окне **Доступность ресурса (обслуж.)**, выберите **Показать матрицу**.  
6. Выберите ресурс для распределения. При выборе можно исходить из соответствия квалификации ресурса уровню, требуемому для выполнения задачи, нахождения его в зоне клиента и/или предпочтений клиента.  
7. Укажите дату, когда ресурс будет иметь в наличии достаточно часов для выполнения задачи, и которая достаточно близка к дате отклика на сервисный заказ.  
8. В поле **Кол-во для распределения** введите число часов, на которое предполагается выделить ресурс для выполнения данной сервисной задачи.  
9. На вкладке **Действия** в группе **Функции** выберите **Распределить**, чтобы распределить выбранный ресурс на выбранную дату.  

     Значение поля **Статус** автоматически устанавливается на **Активно**.  

 Повторите эти шаги для каждой даты, на которую необходимо распределить ресурс по данной сервисной задаче.  

> [!NOTE]  
>  Для сервисного товара в сервисном заказе могут существовать только операции распределения **Активно** с одним ресурсом или группой ресурсов одновременно.  

## <a name="to-allocate-a-resource-using-a-service-order"></a>Распределение ресурса на основе сервисного заказа  
После создания и заполнения сервисного заказа или сервисного предложения можно назначить ресурсы, например, технических специалистов, для выполнения сервисных задач, которые зарегистрированы как строки сервисного товара в документе.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Сервисные заказы**, затем выберите связанную ссылку.  
2. Выберите сервисный заказ, затем выберите **Изменить**.  
3. Выберите строку сервисного товара, соответствующего сервисной задаче, на которую необходимо распределить ресурсы.  
4. Выберите **Распределение ресурсов**.
5. В окне **Распределение ресурсов** выберите неактивную операцию распределения и задачу по сервисному обслуживанию, для которой необходимо выделить ресурс. Если запись распределения не существует, можно создать новую, выбрав **Создать**.  
7. Укажите сервисную задачу, заполнив поле **Код сервисного товара** в той же строке.  
8. В поле **Номер ресурса** выберите ресурс. Если ресурс принадлежит группе ресурсов, код группы ресурсов вводится автоматически в поле **Код группы ресурсов** .  
9. Заполните поля **Дата распределения** и **Распределенные часы**. Поле **Статус** принимает значение **Активно**. Это означает, что ресурс распределен на сервисную задачу.  
10. Если требуется назначить ресурс для всех товаров, выберите **Распределить по всем сервисным товарам**.

> [!NOTE]  
>  Для сервисного товара в сервисном заказе могут существовать только активные операции распределения с одним ресурсом или группой ресурсов одновременно.

## <a name="to-reallocate-resources-on-a-service-order"></a>Чтобы перераспределить ресурсы в сервисном заказе  
Перераспределить ресурсы можно непосредственно из сервисного заказа или сервисного предложения, с которыми в данный момент выполняются действия. Исходная операция все еще существует, но ее статус обновляется следующим образом:  

* Если обслуживание началось, когда распределение имело статус **Активен** (т.е. статус ремонта сервисного товара в операции был изменен на статус **В работе**), то статус распределения меняется с **Требуется перераспределение** на **Завершен**.  
* Если обслуживание не начиналось, когда распределение имело статус **Активна**, статус распределения меняется с **Нужно перераспределение** на **Отменен**.  
* Если происходит перераспределение сервисного заказа, преобразованного из предложения, статус операций распределения, зарегистрированных для предложения, всегда меняется на **Завершен** при перераспределении сервисных товаров сервисного заказа.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Сервисные заказы**, затем выберите связанную ссылку.  
2. Откройте соответствующий сервисный заказ.  
3. Выберите строку сервисного товара, соответствующего сервисной задаче, на которую необходимо распределить ресурсы.  Выберите **Действия**, выберите **Строка**, затем выберите **Распределение ресурсов**.  
4. В окне **Распределение ресурсов** выберите операцию распределения и задачу по сервисному обслуживанию, которой необходимо назначить ресурс. В поле **Номер ресурса** выберите соответствующий ресурс. Если поле уже содержит номер ресурса, он будет перезаписан.  
5. Нажмите клавишу ВВОД. Откроется диалоговое окно, в котором нужно указать, следует ли перераспределить эту операцию. При необходимости заполните поле **Код причины** и нажмите кнопку **Да** для подтверждения перераспределения.  
6. Заполните поля **Дата распределения** и **Распределенные часы**. Теперь операция содержит новый ресурс, и ее статус имеет значение **Активна**.

## <a name="to-reallocate-a-resource-using-the-dispatch-board"></a>Перераспределение ресурса с панели диспетчера  
Если ресурс, распределенный для какой-то сервисной задачи, не может ее выполнить, необходимо произвести перераспределение сервисной задачи. Обычно перераспределение ресурсов по сервисным задачам производится с помощью окна **Панель диспетчера**.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Панель диспетчера**, затем выберите связанную ссылку.  
2. В поле **Фильтр по распределению** выберите **Нужно перераспределение**. Теперь в окне **Панель диспетчера** отображается список тех сервисных заказов, которые содержат сервисные задачи, нуждающиеся в перераспределении.  
3. Выберите соответствующий сервисный заказ. На вкладке **Навигатор** в группе **Планирование** выберите **Распределения ресурсов**. Откроется окно **Распределение ресурсов**.  
4. Выберите операцию распределения с сервисной задачей, на которую нужно перераспределить ресурсы.  
5. В поле **Номер ресурса** выберите соответствующий ресурс. Если поле уже содержит номер ресурса, он будет перезаписан.  
6. Нажмите ENTER. Откроется диалоговое открыто окно **Причины перераспределения операций** с вопросом о необходимости перераспределения этой операции. При необходимости заполните поле **Код причины** и нажмите кнопку **Да** для подтверждения перераспределения.  
7.  Заполните поля **Дата распределения** и **Распределенные часы**. Теперь операция содержит новый ресурс, и ее статус имеет значение **Активна**.  

    > [!NOTE]  
    >  Старая операция все еще существует, но статус обновляется следующим образом:  
    >   
    >  * Если обслуживание началось, когда распределение имело статус **Активен** (т.е. статус ремонта сервисного товара в операции был изменен на статус **В работе**), то статус распределения меняется с **Требуется перераспределение** на **Завершен**.  
    > * Если обслуживание не начиналось, когда распределение имело статус **Активна**, статус распределения меняется с **Нужно перераспределение** на **Отменен**.  
    > * Если происходит перераспределение сервисного заказа, преобразованного из предложения, статус операций распределения, зарегистрированных для предложения, всегда меняется на **Завершен** при перераспределении сервисных товаров сервисного заказа.  

## <a name="to-register-resource-hours"></a>Регистрация часов использования ресурса  
При работе с сервисными товарами в сервисных заказах необходимо регистрировать ресурсо-часы, потраченные на сервис. Ниже приводится процедура регистрации часов использования ресурса в окне **Журнал сервисных товаров**.  

Такую же процедуру можно использовать для регистрации часов в окне **Сервис - строки**, которое можно открыть в окне "Сервисный заказ". Откройте соответствующую сервисную карточку, затем выберите **Действия**, выберите **Заказ**, затем выберите **Сервисные строки**.  

Если один и тот же ресурс работает со всеми сервисными товарами в сервисном заказе, можно зарегистрировать общие ресурсо-часы только для одного товара, а затем разбить строку ресурса, чтобы назначить ресурсо-часы другим сервисным товарам.

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Сервисные задачи**, затем выберите связанную ссылку.
2. Выберите строку, содержащую соответствующий сервисный товар, затем выберите действие **Товарный журнал**.  
3. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

## <a name="to-assign-a-resource-to-all-service-items-in-an-order"></a>Чтобы назначить ресурс для всех сервисных товаров в заказе
Если один и тот же ресурс (например, специалист) работает со всеми сервисными товарами в сервисном заказе, можно зарегистрировать общие ресурсо-часы только для одного товара, а затем разбить строку ресурса, чтобы разделить ресурсо-часы по другим сервисным товарам.  

Следующая процедура показывает, как разбить строки ресурса в окне **Строки сервисного счета**.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Сервисные заказы**, затем выберите связанную ссылку.  
2. Откройте соответствующий сервисный заказ.  
3. На экспресс-вкладке **Строки** выберите **Действия**, выберите **Заказ**, затем выберите **Сервисные строки**. Откроется окно **Сервис - строки**.  
4. Выберите строку ресурса, которую нужно разбить. Содержимое поля **Кол-во** делится между всеми сервисными товарами в заказе.  
5. На вкладке **Действия** выберите действие **Разделить строку ресурса**. Нажмите **Да** для подтверждения.  

    Создаются строки ресурсов для других сервисных товаров в заказе с тем же номером ресурса, что и разбиваемая строка. Количество – это количество в разбиваемой строке, разделённое на число сервисных товаров в заказе.  

## <a name="to-cancel-an-allocation"></a>Отмена распределения  
Можно отменить распределения ресурсов для сервисных задач без перераспределения задач.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Панель диспетчера**, затем выберите связанную ссылку.  
2. Выберите сервисный заказ, затем выберите действие **Распределение ресурсов**.  
3. Выберите операцию распределения, содержащую сервисную задачу, для которой нужно отменить распределение.  
4. Выберите действие **Отменить распределение**.  
5. В поле **Код причины** выберите соответствующий код причины.  
6. Выберите **Да**, чтобы подтвердить отмену.  

  > [!NOTE]  
    > В поле **Статус** параметр **Нужно перераспределение** выбирается автоматически. Если статус ремонта сервисного товара данной операции **Начальный**, он изменяется на **Ссылается**, т. е. обслуживание не начиналось. Если статус ремонта **В работе**, он изменяется на **Частично обслужено** (обслуживание выполнено частично).
