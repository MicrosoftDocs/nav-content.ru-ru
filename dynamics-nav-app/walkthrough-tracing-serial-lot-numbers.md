# <a name="walkthrough-tracing-serial-lot-numbers"></a>Пошаговое руководство. Трассировка серийных номеров и номеров партии
При обнаружении дефектов в продукте следует определить ошибки и запретить отгрузку соответствующих товаров из организации. Если дефектные товары уже отгружены, необходимо выяснить, кому они были отгружены, и, при необходимости, отозвать эти товары.  

Первая задача управления процессом выявления и устранения дефектов заключается в определении источника дефектных товаров и места их использования. Это исследование основывается на исторических данных и облегчается путем поиска среди операций трассировки товаров с помощью окна **Трассировка товаров**.  

Вторая задача управления процессом выявления и устранения дефектов — определить, планируется ли использовать дефектные товары в открытых документах, например неучтенных заказах на продажу или журналах потребления. Эта процедура выполняется в окне **Навигатор**. Можно использовать функцию "Переход" для поиска любых записей базы данных.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве  
В этом пошаговом руководстве показано, как определять дефектные товары, предоставивших их поставщиков, а также места их использования для обеспечения возможности остановки соответствующих заказов или отзыва поставленных товаров.  

В этом пошаговом руководстве рассматриваются следующие задачи:  

-   Трассировка от потребления к источнику  
-   Трассировка от источника к потреблению  
-   поиск всех текущих записей, в которых содержится трассируемый серийный номер или номер партии.  

## <a name="roles"></a>Роли  
В этом пошаговом руководстве демонстрируются задачи, выполняемые исполнителями следующих ролей:  

-   Инспектор по качеству  
-   Заведующий складом  
-   Обработчик заказов  
-   Агент по закупкам  

## <a name="prerequisites"></a>Предварительные требования  
Для выполнения данного пошагового руководства необходимо выполнить следующие действия.  

-   Организация [!INCLUDE[d365fin](includes/d365fin_md.md)].  
-   Для создания новых товаров и нескольких бизнес-транзакций, выполнив шаги в разделе "Подготовка образца данных", далее в этом пошаговом руководстве.  

## <a name="story"></a>Сюжет  
Роман, инспектор по качеству, занимается возвратом проданного товара 1002, шоссейного велосипеда. Клиент, ЗАО «Нефтераз», пожаловался на лопнувший сварной шов на раме. Инженеры по контролю качества подтвердили наличие дефекта в шоссейной раме возвращенного велосипеда. Теперь инспектор по качеству должен определить следующее:  

-   партию бракованных шоссейных рам;  
-   заказ на покупку, по которому была приобретена эта партия.  

В отделе продаж инспектор по качеству узнал, что возвращенному шоссейному велосипеду — товару 1002 — соответствует серийный номер СН1. Используя эти базовые сведения, он должен определить, где в последний раз использовался готовый шоссейный велосипед, а затем выполнить трассировку в обратном направлении до самого раннего источника, чтобы определить номер партии, с которой поступил бракованный компонент — шоссейная рама.  

Результаты этой первой задачи трассировки товара позволят определить дефектные шоссейные рамы и предоставившего их поставщика. Позже, в рамках того же общего процесса трассировки инспектор по качеству должен будет найти все проданные шоссейные велосипеды с шоссейной рамой из бракованной партии, чтобы можно было остановить соответствующие заказы или отозвать товар. Наконец, инспектор по качеству должен найти все открытые документы, в которых используется бракованная партия, чтобы избежать дополнительных транзакций.  

Первые две задачи управления процессом выявления и устранения дефектов выполняются в окне **Трассировка товаров**. Последняя задача выполняется в окне **Навигатор** при интеграции с окном **Трассировка товаров**.  

## <a name="prepare-sample-data"></a>Подготовка образца данных  
Необходимо создать следующие новые товары:  

-   2000, шоссейная рама: трассировка по партии, компонент 1002;  
-   1002, шоссейный велосипед: трассировка по серийному номеру.  

Затем необходимо создать различные транзакции покупки, производства и продажи с этими двумя товарами.

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Товары**, а затем выберите связанную ссылку.

3.  В поле **Номер** введите **2000**, после чего перейдите к заполнению следующих полей.

4.  Все остальные поля содержат приемлемые данные по умолчанию или не требуют заполнения.

14. Верните в поле **Статус** значение **Сертифицировано**.  
15. Нажмите кнопку **OK**, чтобы вставить производственную спецификацию в карточку товара и закрыть окно **Производственная спецификация**.

## <a name="tracing-from-usage-to-origin"></a>Трассировка от потребления к источнику  
 В отделе продаж инспектор по качеству узнает, что серийный номер возвращенного шоссейного велосипеда, товара 1002, равен СН1. Используя этими базовыми сведения, он может определить, где в последний раз использовался готовый шоссейный велосипед, в данном случае — в расходной накладной ЗАО "Нефтегаз". Затем инспектор по качеству должен выполнить трассировку в обратном направлении до самого раннего источника, чтобы определить номер партии, с которой поступила бракованная шоссейная рама, и предоставившего ее поставщика.  

### <a name="to-determine-which-lot-included-the-faulty-frame-and-who-supplied-it"></a>Процедура определения партии, содержащей бракованную раму, и ее поставщика  
1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Трассировка товаров**, затем выберите связанную ссылку.  
2.  В окне **Трассировка товаров** введите в поле **Фильтр по серийному номеру** значение **СН1**, а затем введите в поле **Фильтр по товару** значение **1002**.  
3.  Сохраните значение по умолчанию **Только с трассировкой товара** в поле **Показать компоненты**, а также метод трассировки по умолчанию **Потребление - источник** в поле **Метод трассировки**.  
4.  Выберите действие **Трассировка**.  

    Обратите внимание, что заголовок расходной накладной соответствует критерию поиска. Перед продолжением трассировки убедитесь, что именно по этой расходной накладной ЗАО "Нефтегаз" был поставлен бракованный шоссейный велосипед.  

5.  Выберите строку трассировки, затем выберите действие **Показать документ**.  

    Теперь продолжим трассировку источника расходной накладной на шоссейный велосипед с номером СН1.  
6.  Нажимая значок + в строках трассировки, последовательно разверните цепочку транзакций, ведущую к источнику расходной накладной, и выполните трассировку в обратном направлении.  

    Можно отследить следующую историю транзакций:  

    -   Первым с конца учтенным документом в цепочке транзакций является исходящий учет СН1 из первого запущенного производственного заказа.  
    -   Следующим с конца учтенным документом после этого является учет потребления из первого запущенного производственного заказа. Здесь инспектор по качеству может увидеть, что использовалась шоссейная рама из партии «ПАРТИЯ1».  
    -   Самым первым из учтенных документов в этой цепочке является учтенная приходная накладна, по которой шоссейные рамы из партии «ПАРТИЯ1» поступили в запас.  

    Теперь инспектор по качеству установил, какая из партий шоссейных рам оказалась бракованной, и может найти последнюю строку трассировки, чтобы узнать поставщика, предоставившего эту партию, то есть Custom Metals Incorporated.  

    > [!NOTE]  
    >  Не вносите другие изменения в результаты трассировки при выполнении указаний следующего раздела.  

     На этом первая задача управления процессом выявления и устранения дефектов с помощью окна **Трассировка товаров** завершается. Теперь инспектор по качеству должен определить, не проходили ли шоссейные рамы из партии "ПАРТИЯ1" по другим учтенным документам.  

## <a name="tracing-from-origin-to-usage"></a>Трассировка от источника к потреблению  
 Инспектор по качеству установил, что бракованная шоссейная рама поступила с партией «ПАРТИЯ1». Теперь он должен найти все остальные шоссейные велосипеды с шоссейной рамой из бракованной партии, чтобы можно было остановить поставку или отозвать эти велосипеды.  

 Одним из способов подготовки данного задания трассировки в окне **Трассировка товаров** является ввод значения ПАРТИЯ1 вручную в поле **Фильтр по номеру партии** и значения 2000 в поле **Фильтр по товару**. В этом пошаговом руководстве будет использоваться функция **Обратная трассировка — из строки**.  

### <a name="to-find-all-usage-of-the-faulty-lot"></a>Процедура поиска всех случаев использования бракованной партии  

1.  В окне **Трассировка товаров** выделите строку приходной накладной (последняя строка трассировки), затем выберите **Обратная трассировка - из строки**.  

    Теперь результат трассировки основывается на фильтрах строки трассировки для приходной накладной, партии "ПАРТИЯ1" и товара 2000, а для получения этого результата используется метод трассировки **Источник - потребление**.  

    Чтобы получить представление обо всех случаях использования товара 2000 из партии «ПАРТИЯ1», последовательно разверните все строки трассировки.  

2.  Выберите действие **Развернуть все**.  

    Первые четыре строки трассировки указывают на расходную накладную для компании ЗАО «Нефтераз», проблемы с которой уже устранены. Последняя строка указывает, что по этому же запущенному производственному заказу был произведен еще один шоссейный велосипед, СН2, который затем был продан и отгружен по другой расходной накладной.  

    Инспектор по качеству тотчас же уведомляет отдел продаж о том, что они могут инициировать отзыв дефектного шоссейного велосипеда у клиента, группы «Кэннон».  

    В то же время по последним трем строкам трассировки ему становится ясно, что на основе шоссейных рам из партии «ПАРТИЯ1» было создано два других товара, СН3 и СН4. Он принимает меры для блокирования этих конечных товаров в запасах.  

    На этом вторая задача управления процессом выявления и устранения дефектов с помощью окна **Трассировка товаров**, предназначенного для такого управления, завершается. Поскольку окно **Трассировка товаров** предназначено только для учтенных операций, инспектор по качеству должен продолжить работу в окне **Навигатор** и убедиться, что партия «ПАРТИЯ1» не используется в неучтенных документах.  

## <a name="finding-all-records-of-a-seriallot-number"></a>Поиск всех записей для серийного номера и номера партии  
 Из окна **Трассировка товаров** инспектор по качеству узнал, что партия «ПАРТИЯ1» содержит бракованные шоссейные рамы, определил предоставившего их поставщика, а также учтенные транзакции, в которых они использовались. Теперь он должен определить, присутствует ли партия ПАРТИЯ1 в каком\-либо открытом документе, интегрировав результаты трассировки с окном **Навигатор**, в котором он может выполнить поиск по всем записям базы данных.  

### <a name="to-find-all-occurrences-of-lot1-in-non-posted-records-such-as-open-orders"></a>Процедура поиска всех вхождений партии «ПАРТИЯ1» в неучтенных записях, например открытых заказах  

1.  В окне **Трассировка товаров** выберите первую строку трассировки, приходную накладной партии ПАРТИЯ1.  
2.  Выберите действие **Навигатор**.  

    В окне **Навигатор** предварительно заданы фильтры поиска на основе результатов трассировки для партии «ПАРТИЯ1». Инспектор по качеству определяет, что большинство записей относятся к документам, которые уже были определены в окне **Трассировка товаров**. Например, последняя строка в навигаторе типа «Производственный заказ» указывает на два запущенных производственных заказа, в которых использовались шоссейные рамы из партии «ПАРТИЯ1».  

    Однако вторая строка в навигаторе типа **Строка продажи** является строкой неучтенного документа, поэтому инспектор по качеству продолжает исследование.  

3.  Чтобы открыть запись строки продажи, выберите вторую строку в навигаторе, затем выберите действие **Показать**. Вместо этого можно выбрать значение в поле **Число записей**.  

    Здесь инспектор по качеству может увидеть одну открытую строку продажи для бракованных шоссейных рам. Он тотчас же предлагает отделу продаж отменить этот заказ и инициировать создание нового производственного заказа на основе качественных шоссейных рам.  

 На этом пошаговое руководство по использованию окна **Навигатор** для управления процессом выявления и устранения дефектов при интеграции с окном **Трассировка товаров** завершается.
