# <a name="how-to-plan-for-new-demand-order-by-order"></a>Практическое руководство. Последовательное планирование по каждому заказу
Эта задача планирования может быть выполнена в окне **Планирование заказов**, в котором отображаются все данные нового спроса наряду со сведениями о готовности и предложениями по поставке. В окне обеспечиваются обзор и инструменты, необходимые для эффективного планирования спроса из строк продажи и компонентов и последующего непосредственного создания различных типов заказов поставки.  

В окно **Планирование заказов** можно перейти двумя способами в зависимости от фокуса: из конкретного заказа, для который требуется планирование, или в пакетном режиме, если вы хотите выполнить планирование для всего нового спроса.  


## <a name="to-plan-for-new-production-order-demand"></a>Планирование потребности в новом производственном заказе  
1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Плановые производственные заказы**, затем выберите связанную ссылку. (Эти действия можно выполнять для запланированных, утвержденных или выпущенных производственных заказов.)
2.  Откройте производственный заказ, для которого требуется выполнить планирование, затем выберите действие **Планирование**.  
3.  В окне **Планирование заказов** выберите действие **Вычислить план**.  

Строки планирования отображаются в окне в соответствии с фильтром представления **Производственные требования**, то есть невыполненные строки компонентов всех существующих производственных заказов. Спрос отображается не только для рассматриваемого производственного заказа, так как планировать отдельный производственный заказ без учета требований по потенциально более ранним строкам компонентов было бы "опасно". Планируемые строки рассматриваемого производственного заказа отображаются развернутыми.  

## <a name="to-plan-for-any-new-demand"></a>Планирование с учетом нового спроса  
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Вычислить план**, затем выберите связанную ссылку.  
2.  В окне **Планирование заказов** выберите действие **Вычислить план**.
3.  Нажмите кнопку **Развернуть (+)** перед датой в поле **Дата требования**, чтобы просмотреть подчиненные строки планирования, соответствующие строкам спроса с недостаточным наличием.  
4.  Для каждой развернутой строки планирования (строки спроса) можно просмотреть значения в информационных полях внизу окна.  

    |Параметр|Описанием|  
    |----------------------------------|---------------------------------------|  
    |**Количество на других складах**|Показывает, есть ли товар на другом складе. Можно просмотреть и выбрать его.|  
    |**Наличие заменителей**|Показывает, создана ли для товара замена. Можно просмотреть и выбрать его. Следует заметить, что эта функция применима только к компонентам, то есть, происходящим от строк спроса с типом **Производство**.|  
    |**Доступное количество**|Показывает общую доступность товара, т. е. предполагаемое свободное наличие.|  
    |**Ближайшая доступная дата**|Показывает дату прибытия входящего заказа на поставку, который может покрыть необходимое количество на дату, более позднюю, чем требуемая дата.|  

5.  В поле **Метод пополнения** выберите тип создаваемого заказа на поставку.  

    По умолчанию используется значение из карточки товара или карточки единицы хранения, однако его можно изменить на одно из трех следующих значений:  

    |Параметр|Описанием|  
    |----------------------------------|---------------------------------------|  
    |**Покупка**|Создает заказ покупки.|  
    |**Перемещение**|Создает заказ на перемещение.|  
    |**Произв. заказ**|Создает производственный заказ.|  

    В поле **Поставка от** выберите значение в соответствии с выбранной системой пополнения.  

    > [!NOTE]  
    >  Если это поле не заполнено, при использовании функции **Создание заказов на поставку** система выводит сообщение об ошибке, и заказ на поставку для данной строки планирования не создается. Это, однако, не относится к случаю, когда система пополнения — **Произв. заказ**.  

6.  В поле **Поставка от** выберите в соответствующем списке источник поставки.  

    - Если метод пополнения — **Покупка**, кнопка выбора используется для выбора в окне **Каталог поставщиков товара**.  
    - Если метод пополнения — **Перемещение**, кнопка выбора используется для выбора в окне **Список складов**.  

    Если товар находится на другом складе, в расположенном внизу поле **Количество на другом складе** отображается соответствующее значение, и пользователь может выбрать склад, с которого должен быть получен товар при выполнении заказа на перемещение.  

    Если для требуемого товара существует замена, значением поля **Наличие заменителей** будет **Да**, и пользователь сможет выбрать замену в окне **Замена товара - операции**.  

7.  Установите флажок **Резервировать**, если требуется обеспечить резерв между создаваемым заказом на поставку и строкой спроса, для которой она создается. Это поле по умолчанию не заполнено.  

    > [!NOTE]  
    >  Установить флажок можно, только если значение поля **Резервировать** в карточке товара равно **Необязательно** или **Резервировать**.  

8.  В поле **Кол-во для заказа** можно ввести количество, которое затем будет перенесено в создаваемый заказ на поставку.   
    По умолчанию принимается такое же количество, как и в поле **Требуемое количество**. Но можно заказать больше или меньше данного количества в зависимости от знания о ситуации со спросом. Если, например, в окне **Планирование заказов** имеется несколько несвязанных строк спроса с одной датой выполнения для одного купленного товара, их можно объединить, введя общее необходимое количество в поле **Кол-во для заказа** одной строки, а затем удалив остальные ненужные строки планирования для данного товара.  

9.  В полях **Дата завершения** и **Дата заказа** можно ввести временные параметры создаваемых заказов на поставку.  

    Эти 2 поля взаимосвязаны через поле **Страховой запас времени по умолчанию**, которое находится в окне **Производство - настройка**. По умолчанию срок оплаты совпадает с датой спроса, но при необходимости его можно изменить.  

> [!NOTE]  
>  Если введенная дата наступает после даты требования, программа выдаст предупреждение.  

## <a name="to-make-supply-orders"></a>Создание заказов на поставку  
1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Плановые производственные заказы**, затем выберите связанную ссылку. Эти действия можно выполнять для запланированных, утвержденных или выпущенных производственных заказов.  
2.  Откройте производственный заказ, для которого требуется выполнить планирование, затем выберите действие **Планирование**.  
3.  Установите курсор в нужной строке планирования, затем выберите действие **Заказы на изготовление**.  
4.  В окне **Создание заказов на поставку** на экспресс-вкладке **Планирование заказов** в поле **Заказы на изготовление для** выберите один из следующих параметров.  

    |Параметр|Описанием|  
    |----------------------------------|---------------------------------------|  
    |**Активная строка**|Создайте заказ на поставку только для выбранной строки.|  
    |**Активный заказ**|Создайте заказы на поставку для всех строк в заказе, где установлен курсор.|  
    |**Все строки**|Создайте заказ на поставку для всех строк в окне **Планирование заказов**.|  

5.  На экспресс-вкладке **Параметры** укажите тип заказов на поставку или строк журнала заявок, которые необходимо создать.  

    > [!NOTE]  
    >  Параметры, выбранные в последний раз в окне **Создание заказов на поставку**, сохраняются под кодом пользователя и остаются неизменными до следующего обращения к окну.  

6.  Нажмите кнопку **OK**, чтобы создать предложенные заказы на поставку или строки журнала заявок.  

Теперь, создав соответствующие заказы на поставку, можно распланировать невыполненные требования. Данные конкретных рабочих процессов в окне **Планирование заказов** зависят от внутренних политик организации.  

По завершении процедур планирования в окне **Планирование заказов**, например определения альтернативного способа поставки заданного количества, можно переходить к созданию заказов на поставку для одной или нескольких строк планирования.  

> [!NOTE]  
>  Создаваемые заказы на поставку могут порождать новое зависимое требование, например для подчиненных производственных заказов. Поэтому, прежде чем переходить далее по списку, следует снова выбрать **Вычислить план**, чтобы выявить и разрешить новое требование.
