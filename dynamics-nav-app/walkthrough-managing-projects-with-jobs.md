# <a name="walkthrough-managing-projects-with-jobs"></a>Пошаговое руководство. Управление проектами с помощью работ
В этом пошаговом руководстве даются общие сведения о функциях управления проектами в заданиях. Работы используются для планирования использования ресурсов организации и отслеживания различных затрат, связанных с ресурсами в конкретном проекте. В приложении "Работы" учитывается использование человеко-часов, станко-часов, товаров в запасах и других типов потребления, которые может понадобиться отследить в процессе выполнения работы.  

 В данном пошаговом руководстве описана процедура настройки новой работы в дополнение к заданиям более общего характера, таких как работа с фиксированными ценами, оплата в рассрочку, учет счетов из работ и копирование работ.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве  
 В этом пошаговом руководстве демонстрируются следующие задачи:  

### <a name="setting-up-a-job"></a>Настройка работы  
 Если структура бюджета настроена для работ, создание работы будет простым процессом. В этом пошаговом руководстве рассматриваются следующие процедуры:  

-   Настройка строк рабочего задания и строк планирования.  
-   Создание цен для товаров, ресурсов и счетов Главной книги, характерных для конкретной работы.  
-   выставление счетов из работы.  

### <a name="handling-fixed-prices"></a>Работа с фиксированными ценами  
 В области работ можно применять фиксированные цены и цены на услуги или товары, которые были заранее оговорены клиентами. В этом пошаговом руководстве выполняются следующие задачи:  

-   Узнать о процедуре определения стоимости по контракту и по счету.  
-   Разрешить включение в план работы, по которой не выставлен счет.  

### <a name="copying-a-job"></a>Копирование работы  
 Данная часть пошагового руководства посвящена процедуре полного или частичного копирования работы для сокращения количества вводимых вручную данных и повышения точности. Это включает следующее:  

-   копирование части работы в новую работу;  
-   копирование характерных для работы цен;  
-   копирование строк планирования.  

### <a name="making-payment-by-installment"></a>Оплата в рассрочку  
 Когда работа над большим и дорогостоящим проектом продолжается в течение длительного периода времени, клиенты часто договариваются с организацией про оплату в рассрочку. В этом сценарии демонстрируется процедура оформления оплаты в рассрочку, в том числе:  

-   Создание оплаты в рассрочку для работы.  
-   выставление счетов клиентам;  
-   Учет для использования в работе, настроенной для оплаты в рассрочку.  

## <a name="roles"></a>Роли  
 В этом пошаговом руководстве описываются задачи для следующих ролей:  

-   Менеджер проекта  
-   Член проектной группы  

## <a name="prerequisites"></a>Предварительные требования  
 Перед выполнением заданий этого пошагового руководства необходимо выполнить следующие действия.  

-   Установить демонстрационную базу данных «CRONUS Россия ЗАО».
-   Создать образец данных, выполнив шаги следующего раздела.  

## <a name="story"></a>Сюжет  
Действия этого пошагового руководства разворачиваются вокруг организации «CRONUS International Ltd.», консультационно-конструкторской фирмы, занимающейся проектировкой и монтажом новых инфраструктур, например конференц-залов и офисов с мебелью, вспомогательным оснащением и камерой хранения. Большая часть деятельности этой организации ориентирована на проекты. Станислав — менеджер проекта в CRONUS. Он использует работы, чтобы получить представление о каждой работе, запущенной CRONUS, а также работах, которые уже завершены. Он обычно оформляет сделки с клиентами и вводит основные компоненты работы, которые включают строки задания и планирования, а также цены, в [!INCLUDE[d365fin](includes/d365fin_md.md)]. Он обнаруживает, что создание, поддержка и извлечение сведений не вызывают трудностей. Станиславу также нравится, как [!INCLUDE[d365fin](includes/d365fin_md.md)] позволяет копировать работы и оплату в рассрочку.

 Татьяна, член проектной группы, подотчетной Станиславу, отвечает за ежедневный мониторинг работы. Она вводит данные о своей работе в дополнение к работе, выполненной техническими специалистами по каждому заданию. Она записывает использованные товары и затраты, связанные с их деятельностью.

## <a name="setting-up-a-job"></a>Настройка работы  
 В этом сценарии CRONUS удалось заключить контракт с клиентом, Progressive Home Furnishings, на проектирование конференц-зала и столовой. Клиент находится в США, и для проекта потребуется специальное программное обеспечение. Менеджер проекта должен договориться с клиентом и создать работу, соответствующую условиям этого договора.  

### <a name="to-set-up-a-job"></a>Настройка работы  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите действие **Создать**, чтобы создать новую карточку.  
3.  На экспресс-вкладке **Общее** введите следующие сведения:  

    -   **Описание**: **Рекомендации по оборудованию конференц-зала**  
    -   **Код плательщика**: **01445544**  

4.  На экспресс-вкладке **Учет** введите следующие сведения:  

    -   **Статус**: **Заказ**  
    -   **Работа - учетная группа**: **Настройка**  
    -   **Метод НЗП**: **Себестоимость**  

5.  На экспресс-вкладке **Длительность** введите сегодняшнюю дату в поля **Дата начала** и **Дата окончания**. Эти даты помогут с конвертацией валют при выставлении счета за работу.  
6.  На экспресс-вкладке **Внешняя торговля** задайте в качестве кода валюты значение **USD**. Если в поле **Код валюты счета** выбрано значение USD, счета за работу будут выставляться в долларах США, а планироваться только в локальной валюте организации CRONUS.  

 Можно корректировать цены для клиента для каждого задания в зависимости от соглашений. В следующей процедуре менеджер проекта указывается стоимость времени Татьяны, устанавливает цену для обязательного программного обеспечения и добавляет командировочные расходы, которые клиент согласился оплатить.  

### <a name="to-customize-pricing"></a>Настройка цен  

1.  Из карточки работы выберите действие **Ресурс**.  
2.  В окне **Работа - цены ресурсов** введите следующие сведения:  

    -   **Код**: **Татьяна**  
    -   **Цена единицы**: **20**  

3.  Нажмите кнопку **ОК**, чтобы закрыть окно.  
4.  Выберите действие **Товар**.  
5.  В окне **Работа - цены товаров** введите следующие сведения и настроенную цену:  

    1.  **Код товара**: **80201 (программа обработки графики)**  
    2.  **Цена единицы**: **200**  

6.  Нажмите кнопку **ОК**, чтобы закрыть окно.  
7.  Выберите действие **Счет ГК**.  
8.  В окне **Работа — цены по счету ГК** введите следующую информацию и себестоимость поездки, которую клиент согласился оплатить, + 25 процентов:  

    1.  **Счет ГК**: **8430 (дорога)**  
    2.  **Коэффициент себестоимости единицы**: **1,25**  

9. Нажмите кнопку **ОК**, чтобы закрыть окно.  

 Заключительным этапом настройки задания является добавление рабочих заданий и строк планирования, являющихся частью каждой задачи. От этих строк планирования зависят компоненты, по которым клиенту будет выставлен счет.

    |Номер рабочего задания|Описанием|Тип рабочего задания|  
    |------------------|---------------------------------------|-------------------|  
    |1000|Консультации по оборудованию зала|Сумма (от)|  
    |1010|Консультационная встреча с клиентом|Учет|  
    |1020|Разработка|Учет|  
    |1090|Всего за консультации|Сумма (до)|

 Строка планирования может быть одного из следующих типов:

-   **Контракт**: выставлен счет, но не добавлен в план.

## <a name="calculating-remaining-usage"></a>Расчет остаточного потребления  
 Член проектной группы Татьяна какое-то время занималась определенной работой и должна зарегистрировать затраченное на нее время и потребление. Она не работала больше часов, чем было заранее оговорено с клиентом. Она использует пакетное задание **Расчет остаточного потребления** для вычисления остаточного потребления по работе в журнале. Для каждой задачи при выполнении этого пакетного задания вычисляется разница между запланированным потреблением товаров, ресурсов и ГК расходов и фактическим потреблением, учтенным в операциях книги работ. Оставшееся потребление затем отображается в журнале работ, из которого его можно учесть.  

### <a name="to-calculate-remaining-usage"></a>Процедура расчета остаточного потребления  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Журналы работ**, а затем выберите связанную ссылку.  
2.  В окне **Журнал работ** откройте в поле **Код раздела** список **Журнал работ - разделы**. Выберите раздел журнала работ **Татьяна**.  
3.  Выберите действие **Расчет остаточного потребления**.  
4.  В окне **Работа — расчет остаточного потребления** на экспресс-вкладке **Рабочее задание** выберите поле **Код работы**, а затем выберите нужный номер работы, обычно работу J00010.  
5.  На экспресс-вкладке **Параметры** введите в поле **Номер документа** значение **J00001**. Это упростит дальнейшую трассировку учета.  
6.  Введите в качестве даты учета сегодняшнюю дату.  
7.  Нажмите кнопку **ОК**. В результате будут созданы строки журнала работ, которые являются производными от строк планирования и созданы Станиславом для работы.  
8.  Нажмите кнопку **OK** в окне подтверждения. Созданные строки добавлены в журнал работ.  
9. Убедитесь, что все номера документов равны J00001, затем выберите действие **Учет**. Выберите **Да**, чтобы подтвердить учет.  
10. Теперь строки учтены. Нажмите кнопку **OK**, чтобы закрыть окна.  

## <a name="creating-and-posting-a-job-sales-invoice"></a>Создание и учет счета продажи для работы  
 Далее Татьяна может создать новый счет для работы в целом или части работы. Кроме того, она может присоединять друг к другу счета одного клиента по одной и той же работе. В данном случае выставляется счет на работу в целом, поскольку проект уже завершен.  

### <a name="to-create-a-job-sales-invoice"></a>Процедура создания счета продажи для работы  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите созданную ранее работу, затем выберите действие **Работа — создание счета продажи**.  
3.  На экспресс-вкладке **Рабочее задание** снимите фильтр **Номер рабочего задания**, чтобы выставить счет по работе. В поле **Номер работы** выберите соответствующую работу.  
4.  На экспресс-вкладке **Параметры** введите дату учета и укажите, требуется ли создать счет по каждой задаче или один счет для всех задач.  
5.  Нажмите кнопку **OK**, чтобы создать счет, а затем нажмите кнопку **OK** в окне подтверждения.  

 После того как Татьяна создаст счет, она может получить доступ к нему в модуле **Продажи и маркетинг** в разделе **Обработка заказов** и выполнить дополнительную обработку.  

### <a name="to-post-a-new-sales-invoice"></a>Процедура учета нового счета продажи для работы  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Счета продажи**, а затем выберите связанную ссылку.  
2.  Откройте счет для клиента с номером 01445544. Теперь можно просмотреть сведения, введенные из строк планирования.  
3.  Выберите действие **Учесть**. Выберите **Да**, чтобы подтвердить учет.  

### <a name="to-view-the-posted-invoice"></a>Процедура просмотра учтенного счета  

1.  Откройте работу, затем выберите действие **Строки планирования работ**.  
2.  Выберите любые строки планирования, по которым выставлен счет, затем выберите действие **Счет продажи/кредит-нота**.
3. В окне **Счета работ** выберите действие **Открыть счет продажи/кредит-ноту**.  

 У Татьяны возник вопрос в отношении цен, затрат и прибыли в связи с определенной работой, поэтому она открывает эти сведения в окне **Статистика**.

2.  Выберите действие **Статистика**. Можно просмотреть подробные сведения о цене, себестоимости, и прибыли задачи в местной и в иностранной валюте.  
3.  Нажмите кнопку **Закрыть**, чтобы закрыть окно **Работа - статистика**.  

## <a name="handling-fixed-prices"></a>Работа с фиксированными ценами  
 CRONUS получила контракт на оборудование конференц-зала. Как менеджеру проекта, Станиславу необходимо получить полное представление о заданиях, необходимых для выполнения работы, вместе с бюджетируемой себестоимостью и понесенными издержками по каждому заданию. Кроме того, ему необходимо знать общую цену работы по контракту и сумму, на которую на данный момент был выставлен счет. Он договорился с клиентом о фиксированной цене работы.  

### <a name="to-manage-fixed-pricing-in-jobs"></a>Управление фиксированной ценой в работах  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите номер работы **Водоканал**, затем выберите действие **Строки рабочего задания**.  
3.  Выделите строку 1120, затем в поле **Запланировано (итог. себест.)** щелкните правой кнопкой мыши сумму и нажмите **DrillDown**.  

     Просмотрев строки планирования работ, Станислав определил, что на этом этапе проекта также необходимо 30 часов работы Татьяны. Он договаривается с клиентом о фиксированной цене.  

4.  В окне **Строки рабочего задания** выберите строку 1120, затем выберите действие **Строки планирования работ**.  
5.  Выберите действие **Создать**, чтобы создать новую строку со следующей информацией:  

    -   **Тип строки**: **План и контракт**  
    -   **Тип**: **Ресурс**  
    -   **Номер**: **Татьяна**  
    -   **Кол-во**: **30**;  

7.  Нажмите кнопку **ОК**, чтобы закрыть окно.  
8.  В поле **Запланировано (итог. себест.)** щелкните правой кнопкой мыши поле, а затем еще раз выберите **Раскрытие** в окне **Рабочее задание — строки**. Просмотр изменений в расписании. Как можно увидеть, в план было добавлено 30 часов.  
9. Нажмите кнопку **OK**, чтобы закрыть окна.  

 После добавления Татьяны в план для данной строки задания она занималась этой работой 25 часов. Она вводит эти часы в журнал работ.  

### <a name="to-enter-hours-in-the-job-journal"></a>Процедура ввода часов в журнал работ  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Журналы работ**, а затем выберите связанную ссылку.  
2.  В новой строке введите следующие сведения:  

    -   **Тип строки**: **(пусто)**  
    -   **Дата учета**: **(сегодняшняя дата)**  
    -   **Номер документа**: **J00002**  
    -   **Код работы:**: **водоканал**  
    -   **Номер рабочего задания**: **1120**  
    -   **Тип**: **Ресурс**  
    -   **Номер**: **Татьяна**  
    -   **Кол-во**: **25**;  

3.  Выберите действие **Учесть**.  

     Спустя несколько дней Татьяна проработала еще 10 часов. Теперь общее отработанное ею время составляет 35 часов. Поскольку договор с клиентом был заключен на 30 часов, ему будет выставлен счет только на 5 из этих часов. Татьяна добавит вручную дополнительные пять часов, которые она отработала, в план.  

4.  В окне **Журнал работ** выберите действие **Расчет остаточного потребления**.  
5.  В окне **Работа - расчет остаточного потребления** на экспресс-вкладке **Параметры** введите следующие сведения:  

    -   **Номер документа**: **J00003**  
    -   **Дата учета**: **(сегодняшняя дата)**  

6.  На экспресс-вкладке **Рабочее задание** введите следующие сведения:  

    -   **Код работы:**: **водоканал**  
    -   **Номер рабочего задания**: **1120**  

     Нажмите кнопку **OK** для запуска расчета. У Татьяны осталось 5 часов работы. Поле **Тип строки** пустое, что указывает на то, что остается учесть только потребление, поскольку задание уже запланировано.  

7.  В окне **Журнал работ** создайте новую строку со следующими данными. Убедитесь, что оба номера работ следуют непосредственно за уже использованными номерами:  

    -   **Тип строки**: **План**  
    -   **Код работы:**: **водоканал**  
    -   **Номер рабочего задания**: **1120**  
    -   **Тип**: **Ресурс**  
    -   **Номер**: **Татьяна**  
    -   **Кол-во**: **5**  

     При использовании типа строки **План** обновляются запланированные себестоимость и цены, но себестоимость по контракту и цены, по которым клиенту выставляется счет, остаются прежними.  

8.  Выберите действие **Учесть**. Нажмите кнопку **ОК**, чтобы закрыть окно.  
9. Откройте список **Работы**.  
10. Выберите работу ВОДОКАНАЛ, затем выберите действие **Строки рабочего задания**.  
11. Выделите строку 1120 в поле **Запланировано (итог. себест.)** и щелкните правой кнопкой мыши сумму. Выберите **DrillDown**, чтобы просмотреть сведения.  

     Изменения автоматически вводятся в строку номера рабочего задания 1120. В итоговой себестоимости плановой работы были добавлены в план пять дополнительных часов работы Татьяны.  

12. Выберите кнопку **Закрыть**, чтобы закрыть окно.  
13. Щелкните правой кнопкой мыши сумму в поле **Контракт (итоговая себест.)** и выберите **DrillDown**, чтобы просмотреть сведения.  

     В общую цену по контракту включены только изначально предусмотренные контрактом 30 часов, поскольку именно такое время было оговорено с клиентом.  

## <a name="copying-jobs"></a>Копирование работ  
 Станислав договорился с клиентом, ЗАО «Нефтераз», об оборудовании 10 конференц-залов. Соглашение похоже на более раннее задание. Соответственно, копирование этой более ранней работы позволит сэкономить время.  

 В окне **Копирование работы** можно выбрать работу и строки задания, которые необходимо скопировать. Кроме того, можно выбрать копирование операций книги исходной работы, что приведет к созданию строк планирования на основе фактического потребления, или копирование строк планирования исходной работы, что приведет к копированию в новую работу исходных строк планирования. Затем можно выбрать строку планирования или тип строки книги операций, которые необходимо добавить, выбрав только те данные, которые касаются этой новой работы. И, наконец, можно выбрать работу, в которую необходимо скопировать данные, и указать, следует ли также копировать цены и количества.  

### <a name="to-copy-a-job"></a>Процедура копирования работы  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите действие **Создать**, чтобы создать новую работу. Введите следующие сведения:  

    -   **Описание**: **Оборудование 10 конференц-залов**  
    -   **Код плательщика**: **20000**  

3.  Выберите действие **Копировать рабочие задания из**,  
4.  В окне **Копировать рабочие задания** введите следующие сведения:  

    -   **Код работы:**: **водоканал**  
    -   **С номера рабочего задания**: **1000**  
    -   **Источник**: **Планирование работ - строки**  
    -   **Включая строки планирования типа**: **План + Контракт**  
    -   **В работу с кодом**: **Водоканал Оборудование 10 конференц\-залов**  
    -   Выберите поля **Копировать измерения** и **Копировать количество**.  

5.  Нажмите кнопку **OK**, чтобы скопировать работу, а затем нажмите кнопку **OK**, чтобы закрыть окно подтверждения.  

     Путем сравнения цен, строк рабочих заданий и строк планирования работ для двух работ можно увидеть, что информация была успешно скопирована.  

## <a name="making-payments-by-installments"></a>Оплата в рассрочку  
 CRONUS только что получила крупный проект, на осуществление которого уйдет год. Поскольку для него требуется выделить много ресурсов, менеджер проекта настраивает контракт таким образом, чтобы клиент выплатил часть стоимости авансом, еще часть, когда проект будет наполовину завершен, и окончательно рассчитался после завершения проекта.  

### <a name="to-set-up-a-new-account"></a>Настройка нового счета  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **План счетов**, а затем выберите связанную ссылку.  
2.  В окне **План счетов** выберите действие **Создать**, чтобы создать новую карточку.  
3.  В карточке **нового счета ГК** введите следующие сведения:  

    -   **Номер**: **6630**  
    -   **Название**: **Оплата работы**  

4.  На экспресс-вкладке **Учет** в поле **Общая товарная группа** выберите **ПРОЧЕЕ**. Нажмите кнопку **ОК**, чтобы закрыть окно.  
5.  В окне **План счетов** выберите **Оплата работы № 6630**, затем выберите действие **Создать отступы в плане счетов**. Нажмите **Да** для подтверждения.  

 Следующие процедуры описывают создание нового задания, установки цены и последующей настройки оплаты в рассрочку. В строках рабочего задания можно создать особые строки, предназначенные для оплаты в рассрочку. Все задания, которые будут выполняться в связи с работой, добавленной в план, будет вводиться в строках потребления. Для каждой строки задания оплаты в строках планирования можно ввести в качестве типа строки значение "По контракту", которое означает, что клиенту будет выставлен счет. Введите новую строку для авансового платежа. В строке задания потребления можно ввести сведения об используемых в этом проекте товарах и ресурсах, расширив с их помощью план, например, указав для работы количество часов сотрудника и число используемых товаров.

    -   **Работа - учетная группа**: **Настройка**  
    -   **Метод НЗП**: **Себестоимость**  

3.  На карточке работы выберите действие **Ресурс**. Введите следующие сведения:

    -   **Тип строки**: **По контракту**  
    -   **Дата планирования**: **(сегодняшняя дата)**  
    -   **Тип**: **Счет ГК**  
    -   **Номер**: **6630**  
    -   **Кол-во**: **1**;  
    -   **Цена единицы**: **5000**  

     Нажмите кнопку **ОК**, чтобы закрыть окно.  

7.  В окне **Строки рабочих заданий** выберите **задача 2000**, **Строки планирования работ**, и откройте его.  

     В следующей таблице описываются строки планирования, которые будут созданы.  

    |Линейная.|Тип строки|Дата планирования|Тип|Номер|Количество|  
    |----------|---------------|-------------------|----------|---------|--------------|  
    |1|План|(текущая дата)|Ресурс|Татьяна|120|

     Нажмите кнопку **ОК**, чтобы закрыть окно. В окне **Строки рабочего задания** можно просмотреть обновленные запланированные суммы.

### <a name="to-create-an-invoice"></a>Создание счета
