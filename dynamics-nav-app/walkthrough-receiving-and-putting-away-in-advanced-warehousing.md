# <a name="walkthrough-receiving-and-putting-away-in-advanced-warehouse-configurations"></a>Пошаговое руководство. Получение и складирование в расширенных складских конфигурациях
В [!INCLUDE[d365fin](includes/d365fin_md.md)] входящие процессы по получению и размещению могут быть выполнены четырьмя способами при помощи различных функций в зависимости от уровня сложности склада.  

|Метод|Входящий процесс|Ячейки|Прих. накладные|Складское размещение|Уровень сложности (см. раздел [Сведения о проектировании: настройка склада](design-details-warehouse-setup.md))|  
|------------|---------------------|----------|--------------|----------------|--------------------------------------------------------------------------------------------------------------------|  
|П|Выполните учет приемки и размещения из строки заказа|X|||2|  
|Б|Выполните учет приемки и размещения из документа складского размещения|||X|3|  
|C|Выполните учет приемки и размещения из документа складской приемки||X||4/5/6|  
|D|Выполните учет приемки из документа складской приемки и учет размещения из документа складского размещения||X|X|4/5/6|  

Дополнительные сведения см. в разделе [Сведения о проектировании: входящий складской поток](design-details-inbound-warehouse-flow.md).  

В следующем пошаговом руководстве показан метод D в предыдущей таблице.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве  
В расширенных складских конфигурациях, если настройки склада таковы, что требуется обработка получения в дополнение к обработке размещения, то для регистрации и учета информации о размещении и приемке для входящих документов-источников следует использовать окно **Складская приемка**. При учете складской приходной накладной один или несколько документов складского размещения создаются для указания работникам склада принять полученные товары и разместить их в специальных местах в соответствии с настройкой ячейки или других ячеек. Конкретное размещение товаров отражается при регистрации размещения на складе. Входящим документом-источником может быть заказ на покупку, заказ на возврат продажи, входящий заказ на перемещение, сборочный заказ или производственный заказ, готовая продукция которого готова для размещения. Если приходная накладная создается из входящего заказа, для приходной накладной можно использовать несколько входящих документов-источников. С помощью этого метода можно зарегистрировать множество товаров, поступающих из различных входящих заказов с одной приходной накладной.  

В этом пошаговом руководстве демонстрируются следующие задачи.  

-   Настройка склада БЕЛЫЙ для размещения и получения.  
-   Создание и выпуск двух заказов на покупку для полной обработки склада.  
-   Создание и учет документа складской приемки для нескольких строк заказа на покупку от конкретных поставщиков.  
-   Регистрация складского размещения полученных товаров.  

## <a name="roles"></a>Роли  
В этом пошаговом руководстве демонстрируются задачи, выполняемые исполнителями следующих ролей:  

-   Заведующий складом  
-   Агент по закупкам  
-   Принимающий персонал  
-   Работник склада  

## <a name="prerequisites"></a>Предварительные требования  
Для выполнения данного пошагового руководства необходимо выполнить следующие действия.  

-   Установлен "Cronus Россия ЗАО".  
-   Чтобы назначить себя работником БЕЛОГО склада, выполните следующие действия.  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Сотрудники склада**, а затем выберите связанную ссылку.  
2.  Выберите поле **ИД пользователя** и укажите свою учетную запись пользователя в окне **Пользователи**.  
3.  В поле **Код склада** введите "БЕЛЫЙ".  
4.  Выберите поле **По умолчанию**.  

## <a name="story"></a>Сюжет  
Элен, менеджер склада в CRONUS International Ltd., создает два заказа на покупку принадлежностей у поставщиков 10000 и 20000, которые должны быть доставлены на БЕЛЫЙ склад. Когда товар будет доставлен на склад, Ксения, которая несет ответственность за получение товаров от поставщиков 10000 и 20000, использует фильтр, чтобы создать строки приходной накладной для заказов на покупку от двух поставщиков. Семен учитывает товары, как полученные на склад в одной складской приемке, и обозначает товары, как доступные для продажи или другой потребности. Вадим, работник склада, принимает товары из приемной ячейки и размещает их. Он размещает все единицы в стандартные ячейки, за исключением 40 из 100 полученных шарниров, которые он размещает в отдел сборки путем деления строки размещения. Когда Вадим регистрирует размещение, содержимое ячейки обновляется и товары становятся доступными для подбора со склада.  

## <a name="reviewing-the-white-location-setup"></a>Проверка настройки склада "БЕЛЫЙ"  
Настройка окна **Карточка склада** определяет потоки склада организации.  

### <a name="to-review-the-location-setup"></a>Просмотр настройки склада  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Склады**, а затем выберите связанную ссылку.  
2.  Откройте карточку склада БЕЛЫЙ.  
3.  Убедитесь, что на экспресс-вкладке **Склад** установлен флажок **Расшир. подбор и размещение**.  

    Это означает, что склад настроен для самого высокого уровня сложности, что отражается тем фактом, что выбраны все флажки обработки склада на экспресс-вкладке.  

4.  Обратите внимание, ячейки задаются на экспресс-вкладке **Ячейки** в полях **Код ячейки приемки** и **Код ячейки отгрузки**.  

Это означает, что при создании складской приемки этот код ячейки копируется в заголовок документа складской приемки по умолчанию и в строки получающихся складских размещений.  

## <a name="creating-the-purchase-orders"></a>Создание заказов на покупку  
Заказы на покупку — это наиболее распространенный тип входящего приходного документа.  

### <a name="to-create-the-purchase-orders"></a>Создание заказов на покупку  

1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Заказы на покупку**, затем выберите связанную ссылку.  
2.  Выберите действие **Создать**.  
3.  Создайте заказ на покупку для поставщика 10000 на рабочую дату (23 января) со следующими строками заказа на покупку.  

    |Товар|Код склада|Количество|  
    |----------|-------------------|--------------|  
    |70200|БЕЛЫЙ|100 ШТ|  
    |70201|БЕЛЫЙ|50 ШТ|  

    Перейти к уведомлению склада, что заказ на покупку готов для обработки на складе при поступлении товара.  

4.  Выберите действие **Выпустить**.  

    Перейти к созданию второго заказ на покупку.  

5.  Выберите действие **Создать**.  
6.  Создайте заказ на покупку для поставщика 20000 на рабочую дату со следующими строками заказа на покупку.  

    |Товар|Код склада|Количество|  
    |----------|-------------------|--------------|  
    |70100|БЕЛЫЙ|10 CAN|  
    |70101|БЕЛЫЙ|12 БАН|  

    Выберите действие **Выпустить**.  

    Товары были доставлены от поставщиков 10000 и 20000 на БЕЛЫЙ склад, и Sammy начинает обработку приходных накладных покупки.  

## <a name="receiving-the-items"></a>Получение товаров  
В окне **Складская приемка** можно управлять несколькими входящими заказами ля документов-источников, таких как заказ на покупку.  

### <a name="to-receive-the-items"></a>Получение товаров  
1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Складские приемки**, а затем выберите связанную ссылку.  
2.  Выберите действие **Создать**.  
3.  В поле **Код склада** введите "БЕЛЫЙ".  
4.  Выберите действие **Исп. фильтры для получ. исх. документов**.  
5.  В поле **Код** введите **ACCESSORY**.  
6.  В поле **Описание** введите нужное **Поставщики 10000 и 20000**.  
7.  Выберите действие **Изменить**.  
8.  На экспресс-вкладке **Покупка** в поле **Фильтр по коду поставщика** укажите **10000&#124;20000**.  
9. Выберите действие **Выполнить**. Складская приемка заполняется 4 строками, представляющими строки заказа на покупку для указанных поставщиков. Поле **Кол-во для получения** заполняется, так как не установлен флажок **Не запол. кол-во для обработки** в окне **Фильтры для получения первичных документов**.  
10. При необходимости, если требуется использовать фильтр как описано в начале этого раздела, выберите действие **Получение исходного документа**, затем выберите заказы на покупку у указанного поставщика.  
11. Выберите действие **Учет накладной**, затем выберите кнопку **Да**.  

    Положительные операции в книге товаров создаются с учетом разнесенных приемок принадлежностей от поставщиков 10000 и 20000; товары готовы к размещению в складах из ячейки приемки.  

## <a name="putting-the-items-away"></a>Размещение товаров  
В окне **Складское размещение** можно управлять размещения одновременно для определенного документа складской приемки с охватом составного документов источника. Как и во всех документах складских операций, каждый товар в складском размещении представлен строками "Взять" и "Поместить". В следующей процедуре код ячейки в строках получения товаров соответствует приемной ячейке по умолчанию на складе WHITE, W-08-0001.  

### <a name="to-put-the-items-away"></a>Размещение товаров  
1.  Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Складское размещение**, а затем выберите связанную ссылку.  
2.  Выберите только один документ складского размещения в списке, а затем на вкладке **Главная** в группе **Управление** выберите пункт **Правка**.  

    Откроется документ складского размещения с общим количеством строк "Взять" или "Поместить" 8 шт. для 4 строк заказа на покупку.

    Работнику склада сказано, что для сборочного участка требуется 40 шарниров, и он приступает к разделению единой строки "Поместить" для определения второй строки "Поместить" для ячейки W-02-0001 на сборочном участке, куда он помещает эту часть полученных шарниров.  

3.  Выберите вторую строку в окне **Складское размещение**, строку размещения для товара 70200.  
4.  В поле **Кол-во для обработки** измените значение со 100 на 60.  
5.  На экспресс-вкладке **Строки** выберите **Функции**, а затем — **Разделить строку**. Вставляется новая строка для товара 70200 со значением 40 в поле **Кол-во для обработки**.  
6.  В поле **Код ячейки** введите W-02-0001. Поле **Код зоны** заполняется автоматически.  

    Перейти к регистрации размещения.  

7.  Выберите действие **Зарегистрировать размещение**, затем выберите кнопку **Да**.  

    Полученные принадлежности теперь размещаются в стандартные ячейки товаров, а 40 шарниров помещаются на сборочный участок. Принятые товары теперь доступны для подбора по внутреннему спросу, например для сборочные заказов, или по внешнему спросу, например для расходных накладных продаж.
