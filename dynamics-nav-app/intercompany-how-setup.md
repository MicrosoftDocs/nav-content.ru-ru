# <a name="how-to-set-up-intercompany"></a>Практическое руководство. Настройка межфирменных взаимодействий
Чтобы отправить транзакцию (такую как строка журнала продаж) из одной организации и обеспечить автоматическое создание соответствующей транзакции (такой как строка журнала покупок) в партнерской организации, вовлеченные в этот процесс организации должны согласовать общий план счетов и настроить измерения для использования в межфирменных транзакциях. В качестве межфирменного плана счетов может использоваться, например, упрощенная версия плана счетов родительской организации. Каждая организация сопоставляет свой полный план счетов с общим межфирменным планом счетов, а свои измерения — с межфирменными измерениям.  

Необходимо также настроить код партнера МФ для каждой партнерской организации, которые согласовываются всеми организациями, а затем назначить эти коды карточкам клиентов и поставщиков соответственно, заполнив поле **Код межфирменного партнера**.  

При создании или получении межфирменных строк с товарами можно либо использовать собственные номера товаров, либо настроить для каждого соответствующего товара номера, присвоенные партнером, или в поле **Код товара поставщика**, или в поле **Код общего товара** на карточке товара. Можно также использовать функцию **Перекрестная ссылка товара**. Чтобы сопоставить ваши номера товаров с описаниями товаров ваших межфирменных партнеров, откройте карточку каждого товара, затем выберите действие **Перекрестные ссылки** для настройки перекрестных ссылок между вашими описаниями товара и описаниями межфирменного партнера.  

При создании транзакций межфирменных продаж, включающих ресурсы, для каждого соответствующего ресурса необходимо заполнить поле **Партнер МФ - номер счета ГК покупки** на карточке ресурса. Это номер межфирменного счета главной книги в организации-партнере, в котором учитывается сумма для данного ресурса. Для получения дополнительных сведений см.  

## <a name="to-set-up-companies-for-intercompany-transactions"></a>Настройка организаций для межфирменных транзакций
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Информация об организации**, затем выберите связанную ссылку.  
2. В окне **Информация об организации** заполните поля **Код межфирменного партнера**, **Тип входящих МФ** и **Входящие МФ, подробно**. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

## <a name="to-set-intercompany-partners"></a>Настройка межфирменных партнеров
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Межфирменные партнеры**, затем выберите связанную ссылку.
2. Выберите действие **Создать**.
3. В окне **Межфирменный партнер** заполните требуемые поля.

## <a name="to-set-up-intercompany-vendors-and-intercompany-customers"></a>Настройка межфирменных поставщиков и межфирменных клиентов
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Поставщики**, а затем выберите связанную ссылку.
2. Кроме того, доступ к поставщику можно получить из поля **Код поставщика** в окне **Межфирменный партнер**.
3. Откройте карточку для поставщика, который является межфирменным партнером. Дополнительные сведения см. в разделе [Практическое руководство. Регистрация новых поставщиков](purchasing-how-register-new-vendors.md).
4. В поле **Код межфирменного партнера** выберите соответствующий код межфирменного партнера.
5. Повторите шаги с 1 по 4 для клиентов.

## <a name="to-set-up-intercompany-charts-of-accounts"></a>Настройка межфирменного плана счетов
Чтобы группа организаций могла осуществлять межфирменные транзакции, они должны согласовать план счетов, который будет использоваться как общий эталон. Необходимо согласовать с организациями-партнерами номера счетов, которые вы все будете использовать при создании межфирменных транзакций. Например, родительская организация группы создает упрощенную версию собственного плана счетов, экспортирует этот межфирменный план счетов из своей базы данных в файл XML и распространяет его среди всех организаций группы.  

Если ваша организация является материнской организацией и определяет межфирменный план счетов, который ваша группа будет использовать как общий эталон, следуйте процедуре "Настройка межфирменного плана счетов".  

Если ваша организация является дочерней организацией и получила файл XML, содержащий общий межфирменный план счетов, который ваша группа будет использовать как общий эталон, следуйте процедуре "Импорт межфирменного плана счетов".  

### <a name="to-set-up-the-defining-intercompany-chart-of-accounts"></a>Настройка определяющего межфирменного плана счетов
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **МФ — план счетов**, а затем выберите связанную ссылку.
2. В окне **МФ — план счетов** введите каждый счет в своей строке окна.  
3. Если ваш межфирменный план счетов совпадает с обычным планом счетов или похож на него, можно автоматически заполнить окно, выбрав действие **Копировать из плана счетов**. Новые строки при необходимости можно изменить.

### <a name="to-export-an-intercompany-chart-of-accounts"></a>Экспорт межфирменного плана счетов
Чтобы межфирменные партнеры могли импортировать определяющий план счетов, необходимо экспортировать его в файл.      
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **МФ — план счетов**, а затем выберите связанную ссылку.
2. В окне **МФ — план счетов** выберите действие **Экспорт**, затем выберите кнопку **Сохранить**.
3. Укажите имя и место сохранения файла XML, затем выберите кнопку **Сохранить**.  

### <a name="to-import-the-intercompany-chart-of-accounts"></a>Импорт межфирменного плана счетов  
Когда существует файл для определения межфирменного плана счетов, межфирменные партнеры могут импортировать его, чтобы у них гарантировано были те же самые счета.  
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **МФ — план счетов**, а затем выберите связанную ссылку.  
2. В окне **МФ - план счетов** выберите действие **Импорт**.  
3. Выберите имя и местонахождение файла XML, затем выберите кнопку **Открыть**.  

Окно **МФ - план счетов** заполняется новыми или измененными строками счетов ГК в соответствии с межфирменным планом счетов в файле. Все существующие независимые строки остаются без изменений.

### <a name="to-map-the-intercompany-chart-of-accounts-to-your-companys-chart-of-accounts"></a>Сопоставление межфирменного плана счетов с планом счетов вашей организации  
После определения или импорта межфирменного плана счетов, согласованного для использования вами и вашими партнерами, необходимо связать каждый из межфирменных счетов ГК с одним из счетов ГК своей организации. В окне **МФ - план счетов** укажите, как межфирменные счета ГК во входящих транзакциях будут преобразовываться в счета ГК из плана счетов вашей организации.

Если номера счетов межфирменного плана счетов совпадают с номерами соответствующих счетов плана счетов, то сопоставить счета можно автоматически.

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **МФ — план счетов**, а затем выберите связанную ссылку.  
2. Выберите строки, которые требуется соотнести автоматически, затем выберите действие **Соотнести со счетом с тем же номером**.  
3. Для каждого межфирменного счета в главной книге, который не был сопоставлен автоматически, заполните поле **Соотнесение номера счета ГК**.  

## <a name="to-set-up-default-intercompany-partner-general-ledger-accounts"></a>Настройка счетов главной книги по умолчанию для МФ партнеров  
При создании строки межфирменной продажи или покупки для отправки в качестве исходящей транзакции необходимо ввести счет из межфирменного плана счетов в качестве значения по умолчанию, указывающего, на каком счету партнерской организации учитывается сумма. В окне **План счетов**, для счетов, которые часто используются в строках исходящего межфирменного журнала, можно указать счет ГК по умолчанию для межфирменного партнера. Например, для своих дебиторских счетов можно ввести соответствующие счета к оплате из межфирменного плана счетов.  

Затем, когда вы ввели счет главной книги в поле **Номер баланс. счета** строки межфирменного журнала со значением **Межфирменный партнер** в поле **Тип счета**, поле **Партнер МФ - счет ГК** заполняется автоматически.  

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **План счетов**, а затем выберите связанную ссылку.  
2. В строке для счета ГК, который используется для межфирменных транзакций, в поле **Счет ГК по умолчанию для МФ партнера**, укажите межфирменный счет главной книги, на котором ваш партнер будет выполнять учет, когда вы производите учет на счет главной книги в строке.  
3. Повторите шаг 3 для каждого счета, который часто вводится в поле **Номер баланс. счета** строки межфирменного журнала или документа.

## <a name="to-set-up-intercompany-dimensions"></a>Настройка межфирменных измерений
Если вам и вашим партнерам по межфирменным взаимодействиям нужна возможность обмениваться транзакциями и связанными с ними измерениями, необходимо согласовать измерения, которые будут использоваться всеми партнерами. Например, родительская организация группы создает упрощенную версию собственного набора измерений, экспортирует эти межфирменные измерения из своей базы данных в файл XML и распространяет этот файл среди всех организаций группы. Затем каждый из филиалов импортирует этот файл XML в окно **Межфирменные измерения** и сопоставляет межфирменные измерения с измерениями в своем собственном окне **Измерения**.  

Если ваша организация является материнской организацией и определила набор межфирменных измерений, который ваша группа будет использовать как общий эталон, следуйте процедуре "Определение межфирменных измерений".

Если ваша организация является дочерней и получила файл XML, содержащий межфирменные измерения, которые ваша группа будет использовать как общий эталон, следуйте процедуре "Импорт межфирменных измерений".

### <a name="to-define-the-intercompany-dimensions"></a>Определение межфирменных измерений
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Межфирменные измерения**, затем выберите связанную ссылку.  
2. В окне **Межфирменные измерения** введите каждое измерение в своей строке окна.

    Если ваши межфирменные измерения совпадают с измерениями вашей организации или похожи на них, можно заполнить окно автоматически, используя функцию **Копировать из измерений**, а затем отредактировать полученные строки.  
3. Для экспорта межфирменных измерений в XML-файл для передачи организациям-партнерам, выберите действие **Экспорт**.  
4. Укажите имя и место сохранения файла XML, затем выберите кнопку **Сохранить**.  

### <a name="to-import-the-intercompany-dimensions"></a>Импорт межфирменных измерений  
Когда существует файл для определения межфирменных измерений, межфирменные партнеры могут импортировать его, чтобы у них гарантировано были те же самые измерения.  
1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Межфирменные измерения**, затем выберите связанную ссылку.  
2. В окне **Межфирменные измерения** выберите действие **Импорт**.  
3. Укажите имя и местонахождение файла XML, затем выберите кнопку **Открыть**.  

Импортируются строки в окнах **Межфирменные измерения** и **Значения межфирменных измерений**.  

### <a name="to-map-intercompany-dimensions-to-your-companys-dimensions"></a>Сопоставление межфирменных измерений с измерениями собственной организации
После определения или импорта измерений, которые вы и ваши МФ-партнеры согласились использовать, необходимо связать каждое из межфирменных измерений с одним из измерений вашей организации и наоборот. В окне **Межфирменные измерения** укажите, как межфирменные измерения входящих транзакций будут преобразовываться в измерения из списка измерений вашей организации. В окне **Измерения** укажите, как ваши измерения будут преобразовываться в межфирменные измерения для исходящих транзакций.

Если код какого-то из межфирменных измерений совпадает с кодом соответствующих измерений в списке измерений вашей организации, то можно позволить программе сопоставлять измерения автоматически, затем вы можете сопоставить счета автоматически.

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Межфирменные измерения**, затем выберите связанную ссылку.
2. В окне **Межфирменные измерения** выберите строки, которые требуется соотнести автоматически, затем выберите действие **Соотнести с изм. с тем же кодом**.
3. Для каждого межфирменного измерения, которое не было отображено автоматически, заполните поле **Соотнесение кода измерения**.
4. Выберите действие **Значения межфирменных измерений**.
5. В окне **Значения межфирменных измерений** заполните поле **Соотнесение МФ - код значения измерения**.

    Продолжайте сопоставлять измерения с межфирменными измерениями, выполняя аналогичные шаги.
6. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Измерения**, а затем выберите связанную ссылку.
7. В окне **Измерения** выберите строки, которые требуется соотнести автоматически, затем выберите действие **Соотнести с изм. МФ с тем же кодом**.
8. Для каждого межфирменного измерения, которое не было отображено автоматически, заполните поле **Соответствие коду значения МФ-измерения**.
9. Выберите действие **Значения измерения**.
10. В окне **Значения измерения** заполните поле **Соответствие коду значения МФ-измерения**.
