# <a name="design-details-item-application"></a>Сведения о проектировании: применение товара
При учете транзакции склада учтенное количество записывается в операции журнала товаров, а учет стоимости выполняется в операциях стоимости. Дополнительные сведения см. в разделе [Сведения о проектировании: учет запасов](design-details-inventory-posting.md).  

Кроме того, создается применение товара, чтобы связать получателя затрат и источник затрат для обеспечения перенаправления себестоимости в соответствии с методом учета себестоимости. Дополнительные сведения см. в разделе [Сведения о проектировании: методы учета себестоимости](design-details-costing-methods.md).  

[!INCLUDE[d365fin](includes/d365fin_md.md)] предлагает два вида применения товара.  

|Тип применения|Описанием|  
|----------------------|---------------------------------------|  
|Количество для применения|Создано для всех складских транзакций|  
|Применение себестоимости|Создано для входящих операций с применением количества в результате вмешательства пользователя в особые процессы.|  

Применения товаров можно выполнить следующими способами.  

|Способ|Описанием|Тип применения|  
|------------|---------------------------------------|----------------------|  
|Автоматически|Выполняется при перемещении общей себестоимости в соответствии с методом учета себестоимости.|Количество для применения|  
|Фиксир.|Выполняется пользователем в следующих случаях:<br /><br /> -   Обработка возвратов<br />-   Учет коррекций<br />-   Отмена учета количества<br />-   Создание прямой поставки **Примечание.** Фиксированное применение можно сделать вручную или путем ввода соответствующего номера операции в поле **Примен. из товарной операции** или, например с помощью функции **Получить учт. строки документов для сторнир.**|Количество для применения<br /><br /> Применение себестоимости **Примечание.** Применение себестоимости происходит только во входящих транзакциях, в которых поле **Примен. из товарной операции** заполняется для создания фиксированного применения. См. таблицу ниже.|  

Выполняются ли применения количества или применения себестоимости зависит от направления складской транзакции и от того, выполняется ли применение товара автоматически или является фиксированным в связи со специальными процессами.  

В следующей таблице с учетом центральных полей применения в строках складских транзакций показаны потоки затрат в зависимости от направления транзакции. Также в ней указано, когда и почему применение товара имеет тип количества или себестоимости.  

||Поле "Примен. к товарной операции"|Поле "Примен. из товарной операции"|  
|-|--------------------------------|----------------------------------|  
|Применение для исходящей операции|Исходящая операция извлекает себестоимость из открытой входящей операции<br /><br /> **Количество для применения**|Не поддерживается|  
|Применение для входящей операции|Входящая операция отправляет себестоимость в открытую исходящую операцию.<br /><br /> Входящая операция является источником себестоимости.<br /><br /> **Количество для применения**|Входящая операция извлекает себестоимость из исходящей операции. **Примечание.** При выполнении этого фиксированного применения входящая транзакция обрабатывается как возврат продажи. Поэтому примененная исходящая операция остается открытой. <br /><br /> Входящая операция НЕ является источником себестоимости.<br /><br /> **Применение себестоимости**|  

> [!IMPORTANT]  
>  Фиксировано примененный возврат продажи НЕ считается источником затрат.  
>   
>  Операция продаж остается открытой до учета реального источника.  

Операция применения товара регистрирует следующие сведения.  

|Поле|Описанием|  
|---------------------------------|---------------------------------------|  
|**Номер товарной операции**|Номер товарной операции для транзакции, для которой создана эта операция применения.|  
|**Номер вход. товарной операции**|Номер товарной операции прихода склада, с которой должна быть связана транзакция, если это применимо.|  
|**Номер исход. товарной операции**|Номер товарной операции расхода склада, с которой транзакция должна быть связана, если это применимо.|  
|**Количество**|Применяемое количество.|  
|**Дата учета**|Дата учета транзакции.|  

## <a name="inventory-increase"></a>Приход склада  
При учете прихода склада регистрируется простая операция применения товара без применения к исходящей операции.  

### <a name="example"></a>Пример  
В следующей таблице показана операция применения товара, которая создается при учете приходной накладной на 10 единиц.  

|Дата учета|Номер вход. товарной операции|Номер исход. товарной операции|Количество|Номер товарной операции|  
|------------------|----------------------------------------------|-----------------------------------------------|--------------|---------------------------------------------|  
|01-01-20|1|0|10|1|  

## <a name="inventory-decrease"></a>Расход склада  
При учете расхода склада создается операция применения товара, связывающая расход склада с приходом склада. Эта ссылка создается с использованием метода учета затрат товаров в качестве ориентира. В случае товаров, для которых используется метод учета себестоимости FIFO, "По стандартной и "По средней", связь основывается на принципе "первым пришел — первым обслужен". Расход склада применяется к приходу склада с самой ранней датой учета. В случае товаров, для которых используется метод учета себестоимости LIFO, связь основывается на принципе "последним пришел — первым обслужен". Расход склада применяется к приходу склада с самой последней датой учета.  

В таблице **Операции книги товаров** в поле **Остаток** отображается количество, которое еще не было применено. Если остаток больше 0, устанавливается флажок **Открыто**.  

### <a name="example"></a>Пример  
В следующем примере показана операция применения товара, созданная при учете расходной накладной продажи 5 единиц товара, которые были получены в предыдущем примере. Первая операция применения товара — это приходная накладная покупки. Вторая операция применения — отгрузка продажи.  

В следующей таблице показаны две операции применения товара, являющиеся результатом прихода и расхода склада соответственно.  

|Дата учета|Номер вход. товарной операции|Номер исход. товарной операции|Количество|Номер товарной операции|  
|------------------|----------------------------------------------|-----------------------------------------------|--------------|---------------------------------------------|  
|01-01-20|1|0|10|1|  
|01-03-20|1|2|-5|2|  

## <a name="fixed-application"></a>Фиксированное применение  
Фиксированное применение выполняется, когда указывается, что себестоимость прихода склада должна применяться к конкретному расходу склада или наоборот. Фиксированное применение влияет на оставшиеся количества операций, но фиксированное применение также резервирует точную стоимость исходной операции, к которой или из которой выполняется применение.  

Для выполнения фиксированных применений используются поля **Примен. к товарной операции** или **Примен. из товарной операции** в строках документа, чтобы указать товарную операцию, к которой (или из которой) должна быть применена строка транзакции. Например, можно выполнить фиксированное применение, чтобы создать применение себестоимости, которое указывает, что возврат продажи должен применяться к определенной расходной накладной для точного сторнирования себестоимости по расходной накладной. В этом случае [!INCLUDE[d365fin](includes/d365fin_md.md)] игнорирует метод учета себестоимости и применяет расход склада (или приход, в случае возврата продажи) к указанной пользователем операции книги товаров. Преимущество фиксированного применения состоит в том, что себестоимость по исходной транзакции передается в новую транзакцию.  

### <a name="example--fixed-application-in-purchase-return"></a>Пример — фиксированное применение в возврате покупки  
В следующем примере демонстрируется эффект фиксированного применения возврата покупки товара с использованием метода учета затрат FIFO. В примере иллюстрируется следующий сценарий.  

1. В операции номер 1 пользователь учитывает покупку по цене 10,00 рублей.  
2. В операции номер 2 пользователь учитывает покупку по цене 20,00 рублей.  
3. В операции номер 3 пользователь учитывает возврат покупки. Пользователь выполняет фиксированное применение ко второй покупке, вводя номер операции журнала товаров в поле **Примен. к товарной операции** строки заказа на возврат покупки.  

В следующей таблице показаны операции журнала товаров, полученные по результатам выполнения этого сценария.  

|**Дата учета**|**Тип операции книги товаров**|**Количество**|**Сумма себестоимости (факт.)**|**Номер товарной операции**|  
|----------------------|---------------------------------------------------|------------------|----------------------------------------------------|---------------------------------------------------|  
|01-04-20|Покупка|10|10.00|1|  
|01-05-20|Покупка|10|20.00|2|  
|01-06-20|Покупка (возврат)|-10|-20,00|3|  

Поскольку фиксированное применение основывается на возврате покупки во вторую операцию покупки, товары возвращаются по правильной себестоимости. Если бы пользователь не выполнил фиксированное применение, то значение стоимости возвращенного товара было бы указано неправильно и было бы равно 10,00 рублей, поскольку возврат был бы применен к первой операции покупки по принципу FIFO.  

В следующей таблице показана операция применения товара, являющаяся результатом фиксированного применения.  

|Дата учета|Номер вход. товарной операции|Номер исход. товарной операции|Количество|Номер товарной операции|  
|------------------|----------------------------------------------|-----------------------------------------------|--------------|---------------------------------------------|  
|01-06-20|1|3|10|3|  

Себестоимость второй покупки (20,00 руб.) будет правильно передана возврату покупки.  

### <a name="example--fixed-application-with-average-cost"></a>Пример — фиксированное применение со средней себестоимостью  
В следующем примере демонстрируется эффект фиксированного применения для товара с методом средней себестоимости в следующем сценарии.  

1. В операциях номер 1 и 2 пользователь учитывает два счета покупки. Второй счет имеет неверную прямую себестоимость единицы — 1 000,00 руб.  
2. В операции номер 3 пользователь учитывает кредит-ноту покупки с фиксированным применением, примененным к операции покупки с неправильной прямой себестоимостью единицы. Сумма в поле **Сумма себестоимости (факт.)** для двух фиксированных примененных операций стоимости становится 0,00.  
3. В операции номер 4 пользователь учитывает еще один счет покупки с правильной прямой себестоимостью единицы — 100,00 рублей.  
4. В операции номер 5 пользователь учитывает счет продажи.  
5. Количество инвентаризации — 0, а стоимость инвентаризации также 0,00.  

В следующей таблице показан результат сценария в операциях оценки товара.

|-------------------------------------|-----------------------------------------------|-----------------------------------------|------------------------------------------------|--------------------------------------------|-------------------------------------------------|-----------------------------------------------|----------------------------------|  
|01-01-20|Покупка|1|200.00||Нет|1|1|  
|01-01-20|Покупка|1|1000.00||Нет|2|2|  
|01-01-20|Покупка|-1|-1000|2|Нет|3|3|  
|01-01-20|Покупка|1|100,00||Нет|4|4|  
|01-01-20|Продажа|-2|-300,00||Да|5|5|  

Если бы пользователь не выполнил фиксированное применение между кредит-нотой покупки и покупкой с неправильной прямой себестоимостью единицы (шаг 2 в предыдущем сценарии), то себестоимость была бы скорректирована по-другому.  

В следующей таблице показан результат операций оценки товаров, если шаг 2 в предыдущем сценарии выполняется без фиксированного применения.  

|Дата учета|Тип операции книги товаров|Оцен. кол-во|Сумма себестоимости (факт.)|Примен. к товарной операции|Оценка по средней себест.|Номер товарной операции|Номер операции|  
|-------------------------------------|-----------------------------------------------|-----------------------------------------|------------------------------------------------|--------------------------------------------|-------------------------------------------------|-----------------------------------------------|----------------------------------|  
|01-01-20|Покупка|1|200.00||Нет|1|1|  
|01-01-20|Покупка|1|1000.00||Нет|2|2|  
|01-01-20|Покупка|-1|433,33||Да|3|3|

> [!NOTE]  
>  Если создается фиксированное применение для расхода склада для товара, для которого используется метод учета себестоимости "По средней", расход не будет получать среднюю себестоимость товара как обычно, но вместо этого он получит указанную пользователем себестоимость товара. Этот расход склада больше не будет частью расчета средней себестоимости.

2.  Пользователь учитывает счет продажи.  
3.  Пользователь учитывает кредит-ноту продажи для возвращенного товара, которая применяется к операции продажи, чтобы правильно сторнировать себестоимость.

> [!NOTE]  
>  Если при работе с возвратами или кредит нотами настроено поле **Точный возврат себест. обязат.** в окне **Покупки и кредитор. задолженность - настройка** или в окне **Продажи и дебитор. задолж. - настройка** в соответствии с ситуацией, [!INCLUDE[d365fin](includes/d365fin_md.md)] автоматически заполнит различные поля операций применения при использовании функции **Копировать документ**. Если используется функция **Получить учт. строки документов для сторнир.**, поля всегда заполняются автоматически.  

> [!NOTE]  
>  Если учитывается транзакция с фиксированным применением и операция книги товаров для применение закрыта, то есть остаток по ней нулевой, старое применение автоматически отменяется, и повторно применяется операция книги товаров, используя заданное фиксированное применение.  

## <a name="transfer-application"></a>Применение переноса  
Если товар перемещается с одного склада на другой, внутри складских запасов компании, создается применение между двумя операциями переноса. Оценка операции перемещения зависит от метода учета затрат. В случае товаров, для которых используется метод учета себестоимости "По средней", оценка выполняется с помощью средней себестоимости за период расчета средней себестоимости, в пределах которого находится дата переоценки перемещения. В случае товаров, для которых используются другие методы учета себестоимости, оценка выполняется путем обратной трассировки к себестоимости исходного прихода склада.  

### <a name="example--average-costing-method"></a>Пример — метод средней себестоимости  
В следующем примере, показывающем применение операций переноса, иллюстрируется следующий сценарий для товара с использованием метода учета средней стоимости и периода средних затрат продолжительностью 1 день.  

1. Пользователь покупает товар по цене 10,00 руб.  
2. Пользователь покупает товар снова по цене 20,00 руб.  
3. Пользователь перемещает товар с СИНЕГО в КРАСНЫЙ склад.  

В следующей таблице показано влияние переноса точной стоимости в операциях оценки товара.  

|Дата учета|Тип операции книги товаров|Код склада|Оцен. кол-во|Сумма себестоимости (факт.)|Номер операции|  
|-------------------------------------|-----------------------------------------------|--------------------------------------|-----------------------------------------|------------------------------------------------|----------------------------------|  
|01-01-20|Покупка|СИНИЙ|1|10.00|1|  
|01-01-20|Покупка|СИНИЙ|1|20.00|2|  
|02-01-20|Перемещение|СИНИЙ|-1|15,00|3|  
|02-01-20|Перемещение|КРАСНЫЙ|1|15,00|4|  

### <a name="example--standard-costing-method"></a>Пример. Метод учета себестоимости "По стандартной"  
В следующем примере, показывающем применение операций переноса, иллюстрируется следующий сценарий для товара с использованием стандартного метода учета стоимости и периода средних затрат продолжительностью 1 день.  

1. Пользователь покупает товар по стандартной цене 10,00 руб.  
2. Пользователь перемещает товар из СИНЕГО в КРАСНЫЙ склад по стандартной цене 12,00 рублей.  

В следующей таблице показано влияние переноса точной стоимости в операциях оценки товара.  

|Дата учета|Тип операции книги товаров|Код склада|Оцен. кол-во|Сумма себестоимости (факт.)|Номер операции|  
|-------------------------------------|-----------------------------------------------|--------------------------------------|-----------------------------------------|------------------------------------------------|----------------------------------|  
|01-01-20|Покупка|СИНИЙ|1|10.00|1|  
|02-01-20|Реквизиты|СИНИЙ|-1|10,00|2|  
|02-01-20|Перемещение|КРАСНЫЙ|1|10,00|3|  

Поскольку значение исходного прихода склада составляет 10,00 руб., перемещение оценено по себестоимости, а не по ставке 12,00 руб.  

## <a name="reapplication"></a>Переприменение  
Из-за способа расчета себестоимости единицы товара неправильное применение товаров может привести к отклонениям в средней себестоимости и себестоимости единицы. Следующие сценарии могут стать причиной неверного применения товаров, что требует отмены применений товаров и повторного применения операций журнала товаров.  

* пользователь забыл выполнить фиксированное применение;  
* произведено неправильное фиксированное применение;  
* Необходимо переопределить приложение, созданное автоматически при учете в соответствии с методом учета себестоимости товара.  
* Необходимо вернуть товар, к которому уже вручную была применена продажа, не используя функцию **Получить учт. строки документов для сторнир.**; следовательно, необходимо отменить применение продажи.  

[!INCLUDE[d365fin](includes/d365fin_md.md)] включает функцию анализа и коррекции применений товаров. Эта процедура выполняется в окне **Журнал применения**.
