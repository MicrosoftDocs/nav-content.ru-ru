# <a name="personalizing-your-workspace-in-the-dynamics-windows-client"></a>Персонализация рабочей области в Windows-клиенте Dynamics
Можно персонализировать или настроить рабочую область в соответствии со своей работой и своими предпочтениями, изменяя страницы, чтобы на них отображалась только требуемая информация в тех местах, в которых она нужна. Сделанная вами персонализация изменяет только то, что видите вы, а не то, что видят другие пользователи. Можно персонализировать многие части пользовательского интерфейса (ИП), в том числе отображаемые на ленте действия, положение полей на экспресс-вкладках или информационных панелях и отображаемые на панели навигации пункты меню.

> [!NOTE]  
> Можно также персонализировать страницы с помощью [!INCLUDE[nav_web_md](includes/nav_web_md.md)]. Чтобы узнать, как работает персонализация между двумя клиентами, см. раздел [Обзор персонализации](ui-personalization-overview.md).
 
## <a name="how-to-personalize-your-workspace"></a>Персонализация рабочей области
Большая часть персонализации выполняется с помощью функции **Настройка**, которую можно открыть со страниц практически всех типов, сделав следующее:

1.  Откройте страницу, которую требуется персонализировать.
2.  Слева вверху выберите в меню **Применение** значок ![Кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите один из параметров настройки.

Имеются также некоторые базовые изменения пользовательского интерфейса, такие как настройка размера любого окна или увеличение ширины столбцов, которые производятся непосредственно на странице, а не в рамках пункта **Настроить**.

## <a name="general-information"></a>Общие сведения
При настройке пользовательского интерфейса рекомендуется помнить о следующем: 

-   Можно записать несколько пользовательских настроек одной и той же страницы для разных точек доступа к странице. Например, окно Заказы на продажу можно настроить таким образом, чтобы его внешний вид при открытии из окна Карточка клиента отличался от внешнего вида при открытии из ролевого центра Обработчик заказов на продажу. Точка, из которой осуществляется доступ к настраиваемой странице, записывается в конкретной настройке страницы. Соответственно, может существовать несколько записей настройки страницы в базе данных, как видно в окне **Удаление персонализации пользователя**.

-   Приложение можно настроить для отображения и скрытия элементов пользовательского интерфейса (ИП) (таких как поля, экспресс-вкладки и информационные панели) в зависимости от вашей лицензии и прав доступа. Вы можете просматривать и настраивать только те поля элементов, для которых в вас есть разрешение.

## <a name="customize-ribbons"></a>Настройка лент
Лента обеспечивает доступ к нескольким действиям. Настраивая ленту, можно оптимизировать ее для своих рабочих процессов и предпочтений. Например, если часто используется окно **Измерения**, можно добавить действие **Измерения** в группу действий **Процесс**. Можно также удалить действия, которые никогда не используются, чтобы обеспечить лучшую видимость.  
  
Для настройки лент на страницах можно выполнять следующие задачи:  
  
-   Добавить, переименовать или удалить вкладки, группы, действия и меню.  
-   Измените порядок действий.  
-   Восстановите для ленты настройку по умолчанию.  
  
### <a name="to-customize-a-ribbon"></a>Чтобы настроить ленту
1. Откройте страницу, которую требуется изменить.
2. Слева вверху выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите **Настроить ленту**.

Диалоговое окно **Организовать действия на ленте** разделено на две области. В области **Доступные действия** перечислены все действия, которые можно выбрать для добавления на страницу. В области **Отображать действия в следующем порядке** отображается структура всех действий, которые в настоящее время отображаются на странице.

-   Элементы корневого уровня определяют вкладки.

    -   Элементы второго уровня определяют группу на вкладке.

        -   Элементы третьего уровня определяют меню действий в группе.

### <a name="add-a-group"></a>Добавление группы
Выберите вкладку, на которой должна быть расположена группа, затем выберите **Создать группу**. Нельзя добавить группу в меню.

### <a name="add-a-menu"></a>Добавление меню
Выберите группу, в котором должно быть расположено меню, затем выберите **Создать меню**. Меню можно добавить только в группу или другое меню. 

#### <a name="add-an-action"></a>Добавление действия
Выберите его в области **Доступные действия**, выберите **Добавить**, чтобы добавить его в область **Отображать действия в следующем порядке**, затем кнопками **Сдвинуть вверх** и **Сдвинуть вниз** переместите его в требуемое место.

Нельзя добавить действие на вкладку; только в группу или меню.

###  <a name="limitations-and-recommendations"></a>Ограничения и рекомендации 
Помните о следующих ограничениях при настройке ленты:  
  
-   Системные вкладки или группы, такие как **Главная** или **Создать**, нельзя переместить или удалить. Позиция некоторых групп, таких как **Новый документ** является неизменной.  
-   Действия или группы с динамической видимостью нельзя добавить или удалить. 
-   Можно создавать меню только внутри групп, но не вкладок.  
-   Можно вложить меню в другое меню, однако это не рекомендуется.  
-   Если наблюдается непредвиденное поведение групп и действий после настройки ленты выполните следующие действия:  
    
    1.  Опустошите, однако не удаляйте группу, в которой возникла проблема.  
    2.  Закройте диалоговое окно с помощью кнопки **ОК**.  
    3.  Откройте диалоговое окно еще раз и повторно добавьте действия в группу.  

> [!IMPORTANT]  
>  Любая настройка, которая приводит к изменению ленты, может повлиять на инструкции, предоставляемые в справке [!INCLUDE[navnow_md](includes/navnow_md.md)], поскольку шаги навигации в справке могут указывать на отличный макет ленты.

## <a name="customize-fasttabs"></a>Настройка экспресс-вкладки
Экспресс-вкладки помогают организовать информацию о страницах в простые управляемые группы. Экспресс-вкладки можно настроить на страницах таким образом, чтобы они поддерживали ваши рабочие процессы. Например, может потребоваться отобразить меньше экспресс-вкладок или скрыть определенные поля на экспресс-вкладках. Можно также активировать отображение наиболее важных полей в заголовках экспресс-вкладок, когда данные экспресс-вкладки свернуты.  

### <a name="to-customize-a-fasttab"></a>Настройка экспресс-вкладки  
  
1.  Откройте страницу, которую требуется изменить.
2.  Выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите **Настроить эту страницу**.  
3.  В диалоговом окне **Настроить <Page Name>** выберите **Экспресс-вкладки**.  
  
### <a name="add-move-or-remove-fasttabs"></a>Добавление, перемещение или удаление экспресс-вкладок
Поле **Показать экспресс-вкладки в следующем порядке** содержит экспресс-вкладки, которые в текущий момент имеются на странице, и порядок, в котором они отображаются. С помощью кнопок **Добавить**, **Удалить**, **Сдвинуть вверх** и **Сдвинуть вниз** сделайте изменения.

### <a name="show-and-hide-fields-on-fasttabs"></a>Отображение и скрытие полей на экспресс-вкладках
В поле **Показать экспресс-вкладки в следующем порядке** выберите экспресс-вкладку, для которой необходимо внести изменение, и щелкните **Настроить экспресс-вкладку**. С помощью кнопок настройте поля, которые должны отображаться, и их порядок на странице.

Установите **Важность** следующим образом:
-   Если требуется, чтобы поле отображалось в заголовке экспресс-вкладки, когда она свернута, установите значение **Повышенная**.
- Если необходимо, чтобы поле было скрыто, если только пользователь не выбрал действие **Показать больше полей** на экспресс-вкладке, установите значение **Дополнительно**.
-   **Стандартная** — это настройка по умолчанию или нормальная настройка.

### <a name="set-up-field-for-quick-entry"></a>Настройка поля для экспресс-ввода 
Установите флажок **Экспресс-ввод** для добавления поля в список полей экспресс-ввода. Если при работе на странице нажать клавишу ВВОД в поле, указатель переходит в следующее поле, которое задано как поле экспресс-ввода. 

## <a name="customize-factboxes"></a>Настройка информационных панелей
На информационных панелях отображаются сведения, касающиеся записи, выделенной в списке или открытой на странице задач. Можно выбрать информационные панели, которые должны отображаться в области информационных панелей. На информационных панелях можно также настроить отображение только тех полей, которые вам необходимы.  
  
### <a name="to-show-or-hide-the-factbox-pane"></a>Скрытие или отображение области информационных панелей
Информационные панели содержатся в области информационных панелей, которую можно отобразить или скрыть отдельно для каждой страницы. Это позволяет легко скрыть несколько информационных панелей без необходимости удалять их по отдельности. 

1.  Откройте страницу, которую требуется изменить. 
2.  Выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите **Информационные панели**. Флажок означает, что область информационных панелей отображается.  

### <a name="to-customize-the-factbox-pane"></a>Настройка области информационных панелей  
1.  Откройте страницу, которую требуется изменить. 
2.  Выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите **Выбрать информационные панели**.  
    
### <a name="add-a-factbox"></a>Добавление информационной панели  
  
Выберите информационную панель, которую требуется добавить в область информационных панелей, в поле **Доступные информационные панели**, затем выберите кнопку **Добавить**.  
  
### <a name="remove-a-factbox"></a>Удаление информационной панели  
  
В поле **Показать информационные панели в следующем порядке** выберите информационные панели, затем кнопку **Удалить**.   
  
### <a name="change-the-order-of-the-factboxes"></a>Изменение порядка информационных панелей  
  
В поле **Показать информационные панели в следующем порядке** выберите информационную панель, которую необходимо переместить, затем с помощью кнопок **Сдвинуть вверх** и **Сдвинуть вниз** расположите ее в нужном месте.  
  
### <a name="change-the-fields-in-a-factbox"></a>Изменение полей на информационной панели  
  
1.  В поле **Показать информационные панели в следующем порядке** выберите информационную панель, затем выберите **Настроить часть**.  
  
2.   В поле **Доступные поля** перечислены все поля, доступные для выбора. В поле **Отображаемые поля** указаны все поля, которые в текущий момент отображаются на информационной панели. Используйте кнопки для добавления, удаления и перемещения полей.   


## <a name="customize-columns-in-a-list-or-on-document-lines"></a>Настройка столбцов в списке или строках документов
Для получения более полного представления о необходимой информации можно настроить страницы списка и карточек, добавляя или удаляя столбцы в сетках, изменяя порядок столбцов и добавляя область закрепления.  
  
### <a name="to-add-remove-and-arrange-columns"></a>Для добавления, удаления или изменения порядка столбцов  
  
1.  Столбцы можно добавлять, удалять или изменять их порядок двумя способами:

    -   Выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем **Выбрать столбцы**.
    -   Щелкните правой кнопкой мыши заголовок столбца, затем выберите пункт **Выбрать столбцы**.

2.  В диалоговом окне **Выбрать столбцы для отображения на странице** область **Доступные столбцы** содержит скрытые столбцы. Область **Показать столбцы в следующем порядке** содержит отображаемые столбцы. Для перемещения столбцов из одного поля в другое воспользуйтесь кнопками **Добавить** и **Удалить**. С помощью кнопок **Вверх** и **Вниз** задайте положение столбцов. 

>[!TIP]
>Установите флажок **Экспресс-ввод** для добавления поля в список полей экспресс-ввода. Если при работе на странице нажать клавишу ВВОД в поле, указатель переходит в следующее поле, которое задано как поле экспресс-ввода. 

### <a name="to-set-the-freeze-pane"></a>Для задания области закрепления
Список можно содержать несколько столбцов, в результате чего для просмотра всех столбцов может требоваться прокрутка по горизонтали. Могут иметься несколько столбцов, которые должны всегда отображаться, даже при прокрутке. Для этого можно добавить вертикальную область закрепления, чтобы запретить прокручивание некоторых столбцов. Это дает возможность обеспечить, чтобы при прокрутке перемещались только менее важные столбцы.

Чтобы настроить область закрепления, выберите столбец, после которого должна начинаться область закрепления, затем выберите **Добавить область закрепления**.

## <a name="customizing-the-navigation-pane"></a>Настройка области навигации
В области навигации отображается меню ссылок на различные страницы списка. Ссылки группируются под кнопками корневого уровня. 

### <a name="to-customize-the-navigation-pane"></a>Настройка панели навигации  
  
Выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите **Настроить панель навигации**.  
  
### <a name="rename-or-rearrange-buttons-in-the-navigation-pane"></a>Переименование или изменение расположения кнопок на панели навигации  
В диалоговом окне **Настройка области навигации** выделите в области слева кнопку, которую необходимо переместить, переименовать или удалить, а затем нажмите соответствующую кнопку в центре окна.

Кнопку **В начало** нельзя перемещать, переименовывать или удалять. Кнопку **Подразделения** можно удалить из панели навигации, но невозможно переименовать или переместить. 
  
### <a name="add-a-new-menu-button"></a>Добавление новой кнопки меню 
Можно создать новую кнопку корневого уровня, затем добавить меню ссылок под кнопкой для открытия других страниц.

1. В диалоговом окне **Настройка области навигации** выберите **Создать**, затем введите название в поле **Название**.
2.  Нажмите кнопку **ОК**.

Можно добавить ссылки к кнопке.  
  
### <a name="add-a-link-to-a-button"></a>Добавление ссылки к кнопке   
При наличии прав на просмотр списка, например списка заказов на продажу, можно добавить ссылку на этот список из кнопки в области навигации.  
  
1.  В диалоговом окне **Настройка области навигации** в поле **Кнопки области навигации** выберите меню, в которое необходимо добавить ссылку.  
  
2.  Нажмите кнопку **Добавить**.  
  
3.  Перейдите к ссылке, которую требуется добавить, затем нажмите кнопку **OK**.  
> [!TIP]
> Если какая-либо ссылка присутствует на странице **Подразделения**, ее также можно добавить в область навигации. Дополнительные сведения см. в разделе "Добавление ссылок со страницы «Подразделения» в ролевой центр".  
  
### <a name="move-or-copy-a-link-from-one-button-to-another"></a>Перемещение или копирование ссылки из одной кнопки в другую  
  
1.  В диалоговом окне **Настройка области навигации** в поле **Кнопки области навигации** выберите меню, в котором в данный момент отображается ссылка.  
  
2.  В области **Списки** выберите ссылку, которую необходимо переместить, а затем выберите **Переместить** или **Копировать в**.  
  
3.  Выберите кнопку навигации, в которую необходимо добавить ссылку, затем нажмите кнопку **OK**.  
  
### <a name="rearrange-the-order-of-a-links-under-a-button"></a>Изменение порядка ссылок под кнопкой  
  
1.  В области **Списки** выберите ссылку, которую необходимо переместить.  
  
2.  С помощью кнопок **Вверх** и **Вниз** задайте положение ссылки.

## <a name="adding-department-links-to-the-role-center"></a>Добавление ссылок со страницы "Подразделения" в ролевой центр
Иногда требуется, чтобы некоторые ссылки на странице **Подразделения** отображались также в ролевом центре для удобства доступа. Возможность добавления ссылки в ролевой центр зависит от категории ссылки на странице **Подразделения**.

В следующей таблице описываются типы ссылок, встречающиеся в каждой категории на странице **Подразделения**, а также места в ролевом центре, в которые их можно добавить.  
  
|**Категория**|**Содержимое**|**Место добавления ссылки**|  
|------------------|------------------|---------------------|  
|Списки|Страницы списка|Кнопка **Главная** на панели навигации|  
|Задачи|Страницы задач, пакетные задания, журналы|Вкладка **Действия** на ленте|  
|Отчеты и анализ|Отчеты, пакетные задания, окна матриц|Вкладка **Отчеты** на ленте|  
|Документы|Документы, например счета и напоминания|**Отчеты** на ленте|  
|Архив/История|Учтенные или завершенные документы, регистры|Кнопка **Главная** на панели навигации|  
|Администрация|Окна настройки|Вкладка **Действия** на ленте|  
  
### <a name="to-copy-department-links-to-your-role-center"></a>Копирование ссылок подразделений в ролевой центр  
  
1.  Откройте страницу **Подразделения**.  
  
2.  Щелкните ссылку правой кнопкой, затем выберите одно из следующих действий (будет доступно только одно из них).  
  
    |**Действие**|**Место добавления ссылки**|  
    |----------------|----------------------------|  
    |**Добавить в область навигации**|Кнопка **Главная** в области навигации ролевого центра.|  
    |**Добавить в "Действия" на ленте ролевого центра**|Меню **Действия** в ленте ролевого центра|  
    |**Добавить в Отчеты на ленте ролевого центра**|Меню **Отчеты** на ленте ролевого центра|  
  
3.  Подтвердите отобразившее сообщение.  
  
 Ссылка появится в меню, в которое она была добавлена. Однако может возникнуть необходимость в перемещении этой связи в другое положение в пределах меню. Например, при добавлении ссылки в область навигации она будет отображаться в меню **В начало**, однако ее можно переместить в другое меню в области навигации. Дополнительные сведения см. в разделе "Настройка области навигации". 

## <a name="adding-charts-to-role-centers-and-list-pages"></a>Добавление диаграмм в ролевые центры и на страницы списка
При наличии сложной информации может потребоваться получить визуальное представление данных для выявления тенденций и принятия решений. Например, может потребоваться мониторинг сальдо каждого банковского счета организации в виде схемы. Панель диаграмм используется для визуального отображения данных из списка в следующих типах страниц:  
  
-   В ролевом центре, в котором можно выбрать из предопределенных универсальных диаграмм.  
  
-   На странице списка можно выбрать просмотр списка в виде диаграммы.  
  
### <a name="to-add-a-generic-chart-to-your-role-center"></a>Добавление универсальной диаграммы в ролевой центр  
  
1.  В ролевом центре выберите в меню **Применение** значок ![кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить**, затем выберите **Настроить эту страницу**.  
  
2.  В окне **Настройка ролевого центра** в поле **Доступные части** выберите **Часть диаграммы** и нажмите кнопку **Добавить**.  
  
3.  С помощью кнопок **Вверх**, **Вниз**, **Влево** и **Вправо** расположите часть диаграммы в ролевом центре.  
  
4.  Выберите часть диаграммы, выберите **Настроить часть**.  
  
5.  В окне **Настройка диаграммы** выберите предварительно определенную диаграмму для отображения и нажмите кнопку **OK**.  
  
### <a name="to-view-a-list-as-a-chart"></a>Просмотр списка как диаграммы  
  
1.  На странице списка выберите действие **Показать как диаграмму**.  
  
2.  Выберите единицу измерения и размерность, чтобы создать пользовательскую диаграмму. Чтобы просмотреть дополнительную информацию, следует выделить вторичное измерение. Например, выбрав измерение по оси X и **Число измерений** по оси Y можно создать простую гистограмму.  
  
> [!NOTE]  
>  По умолчанию область диаграмм скрыта, поскольку ее отображение может вызвать падение производительности. Отображать диаграмму следует только при необходимости получения информации.  
    
## <a name="handling-external-files-and-automation-objects"></a>Обработка внешних файлов и объектов автоматизации
Если [!INCLUDE[navnow_md](includes/navnow_md.md)] получает внешний файл, открывается диалоговое окно. В дополнение к выбору возможных операций с файлом можно решить, как будет обрабатываться данный тип файла при следующем получении.  
  
Если [!INCLUDE[navnow_md](includes/navnow_md.md)] требуется для работы с объектом автоматизации, открывается диалоговое окно. Можно определить, должен ли тип объекта быть всегда разрешен или запрещен к выполнению.  
  
### <a name="to-specify-how-to-handle-external-files"></a>Указание способа обработки внешних файлов  
  
1.  Если появилось диалоговое окно, снимите флажок **Всегда спрашивать перед открытием файлов этого типа**, если вы хотите, чтобы [!INCLUDE[navnow_md](includes/navnow_md.md)] запомнил параметр, выбранный на шаге 2. Когда в следующий раз нужно будет обработать тип файла, диалоговое окно не появится, и файл будет обработан, как указано на шаге 2.  
  
     В качестве альтернативы установите флажок **Всегда спрашивать перед открытием файлов этого типа**, чтобы всегда открывалось диалоговое окно при получении файлов этого типа.  
  
2.  Щелкните **Открыть**, **Сохранить** или **Отмена**. Файл обрабатывается в соответствии с выбранными параметрами.  
  
### <a name="to-specify-how-to-handle-automation-objects"></a>Указание способа обработки объектов автоматизации  
  
Если открывается диалоговое окно, установите флажок **Всегда разрешать**, если требуется, чтобы [!INCLUDE[navnow_md](includes/navnow_md.md)] всегда выполнял этот тип объекта автоматизации. При следующей попытке запуска этого типа объекта автоматизации данное диалоговое окно не откроется и объект автоматизации запустится напрямую.  
  
В качестве альтернативы установите флажок **Всегда запрещать**. При следующей попытке запуска этого типа объекта автоматизации данное диалоговое окно не откроется и объект автоматизации не запустится.  

## <a name="CancelPersonalization"></a>Отмена персонализации
Отмену персонализации можно разделить на две категории:

-   Отмена изменений, сделанных с помощью функции **Настроить**.
-   Отменять изменений базового пользовательского интерфейса. 

### <a name="cancel-customization"></a>Отмена настройки
Если требуется отменить все настройки пользовательского интерфейса, которые были заданы для страницы в текущем сеансе входа или с момента последней отмены настроек пользовательского интерфейса, можно использовать окно **Удаление персонализации пользователя**. Макет страницы, для которой удаляется персонализация, сбрасывается до конфигурации по умолчанию для вашего профиля.  
  
Если требуется отменить только настройки пользовательского интерфейса, заданные для определенной области пользовательского интерфейса, например ленты, можно нажать кнопку **Восстановить значения по умолчанию** в окне **Настроить**. Макет конкретной области ИП на этой страницы сбрасывается до конфигурации по умолчанию для вашего профиля.  
  
#### <a name="to-cancel-all-ui-customization-that-you-have-made-to-a-page"></a>Отмена всех настроек ИП, сделанных для страницы  
  
1.  В поле **Поиск** введите **Удаление персонализации пользователя**, затем выберите соответствующую ссылку.  
  
2.  Выберите страницу, для которой требуется отменить настройки пользовательского интерфейса, а затем на вкладке **Главная**, в группе **Вид** выберите **Удалить**.  
  
> [!NOTE]  
>  Все настройки пользовательского интерфейса страницы, выполненные пользователем в текущем сеансе входа или с момента последнего использования окна **Удаление персонализации пользователя**, отменяются. Макет страницы сбрасывается до конфигурации по умолчанию для вашего профиля в соответствии с настройками администратора или особенностями установки с Microsoft Dynamics NAV.  
  
#### <a name="to-cancel-ui-customization-that-you-have-made-to-a-ui-area-on-a-page"></a>Отмена настройки ИП, сделанной в области ИП на странице  
  
1.  На странице, на которой была выполнена настройка области пользовательского интерфейса, например лента, выберите в меню **Применение** значок ![Кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon"), выберите **Настроить** и щелкните **Настроить <UI area>**.  
  
2.  В нижней части окна **Настроить** нажмите кнопку **Восстановить значения по умолчанию**.  
  
> [!NOTE]  
>  Все настройки области пользовательского интерфейса, выполненные пользователем для этой страницы в текущем сеансе входа или с момента последнего нажатия кнопки **Восстановить значения по умолчанию**, отменяются. Макет области пользовательского интерфейса сбрасывается до конфигурации по умолчанию для вашего профиля в соответствии с настройками администратора или особенностями установки с Microsoft Dynamics NAV.

### <a name="cancel-basic-ui-changes"></a>Отмена изменений базового пользовательского интерфейса
Можно отменить базовые изменения ИП, открыв окно **Сброс настроек, заданных пользователем**, из ролевого центра.  
  
Изменения базового пользовательского интерфейса включают следующее:
 -  Изменение размера и положения любого из окон.
 -  Изменение ширины столбцов
 -  Изменение высоты заголовков столбцов.
 -  Сортировка столбцов в списке.
 -  Отображение списков в виде диаграмм.
 -  Определение метода обработки внешних файлов и объектов автоматизации.  
 
#### <a name="to-cancel-basic-ui-changes"></a>Отмена изменений базового пользовательского интерфейса
  
1.  Перейдите в ролевой центр.  
  
     В меню **Применение** ![Кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon") выберите **Настроить**, затем выберите **Сброс настроек, заданных пользователем**.  
  
2.  Нажмите кнопку **Сброс настроек пользовательского интерфейса**. В качестве альтернативы нажмите кнопку **Сбросить все**, чтобы отменить решения по обработке файлов и объектов автоматизации.  
  
 Все базовые изменения пользовательского интерфейса, выполненные пользователем в текущем сеансе входа в [!INCLUDE[navnow_md](includes/navnow_md.md)] или с момента последнего нажатия кнопки **Сброс настроек пользовательского интерфейса**, отменяются. Пользовательский интерфейс сбрасывается до конфигурации по умолчанию для вашего профиля.  
  
#### <a name="to-cancel-your-decision-for-running-or-saving-external-files"></a>Отмена решения по запуску и сохранению внешних файлов  
  
1.  Перейдите в ролевой центр.  
  
     В меню **Применение** ![Кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon") выберите **Настроить**, затем выберите **Сброс настроек, заданных пользователем**.  
  
2.  Нажмите кнопку **Сброс решения по обработке файлов**. В качестве альтернативы нажмите кнопку **Сбросить все**, чтобы отменить просмотр изменений и решение по обработке файлов и объектов автоматизации.  
  
 Все решения по обработке типов файлов по умолчанию, принятые пользователем в текущем сеансе входа или с момента последнего нажатия кнопки **Доступ к файлу клиента**, отменяются, и выполняется сброс настроек профиля до конфигурации по умолчанию. В следующий раз когда [!INCLUDE[navnow_md](includes/navnow_md.md)] получает внешний файл любого типа, открывается диалоговое окно с параметрами **Сохранить**, **Выполнить** и **Отмена**.  
  
### <a name="to-cancel-your-decision-for-handling-automation-objects"></a>Отмена решения по обработке объектов автоматизации  
  
1.  Перейдите в ролевой центр.  
  
     В меню **Применение** ![Кнопка меню "Применение" в строке меню](media/applicationmenuicon.png "ApplicationMenuIcon") выберите **Настроить**, затем выберите **Сброс настроек, заданных пользователем**.  
  
2.  Нажмите кнопку **Сброс решений по автоматизации**. В качестве альтернативы нажмите кнопку **Сбросить все**, чтобы отменить просмотр изменений и решение по запуску или сохранению внешних файлов.  
  
 Все решения о запуске объектов автоматизации, принятые пользователем в текущем сеансе входа или с момента последнего нажатия кнопки **Сброс решений по автоматизации**, отменяются. Поведение обработки файлов сбрасывается до конфигурации по умолчанию для вашего профиля. В следующий раз [!INCLUDE[navnow_md](includes/navnow_md.md)] обязан будет выполнить объект автоматизации любого типа. Пользователю открывается диалоговое окно с параметрами **Всегда разрешать** и **Никогда не разрешать**.
